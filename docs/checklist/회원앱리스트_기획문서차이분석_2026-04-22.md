# 회원앱리스트 vs 회원앱 기획문서 차이 분석

## 1. 결론

- 현재 기준에서 회원앱 체크리스트 37개 항목은 기획 문서에 모두 반영되도록 정리했다.
- 현재 프로젝트 퍼블리싱 기준으로는 `구현 33 / 부분구현 4 / 미구현 0` 상태다.
- 일부 항목은 `용어 통일` 또는 `정책 해석`이 들어갔다.
- 핵심 해석은 아래 두 가지다.
  - `포인트` → `마일리지`
  - `회원가입` → `CRM 등록 + 모바일 앱 연동`

즉, 지금 차이점의 중심은 `누락`이 아니라 `기획 해석 방식`과 `운영 후속 범위`다.

---

## 2. 차이 유형

### 2.1 동일 반영

아래 항목은 체크리스트 의미와 기획 문서 표현이 거의 동일하다.

- 로그인
- PT/GX/Golf 예약 흐름
- 골프 강사 예약
- 강사 상세
- 대기 예약
- 수업 후기
- 인바디/FMS
- 상품 구매 / 결제 / 영수증
- 개인 결제
- 알림센터
- 온보딩 설문
- 첫 루틴 제안
- 재등록 추천 플랜
- 약관 / 동의관리 / 회원 탈퇴

### 2.2 용어 통일 반영

| 체크리스트 표현 | 기획 문서 표현 | 해석 |
|---|---|---|
| 포인트 | 마일리지 | 동일 리워드 단위 |
| 포인트 사용 | 마일리지 사용 | 결제 리워드 사용 |
| 회원가입 | CRM 등록 + 앱 연동 | 가입 정책 차이 반영 |

### 2.3 상태 분리 반영

아래 항목은 기획 문서에 포함하지만, 현재 구현 상태는 별도 표기로 분리했다.

- PT 예약의 OT1/OT2 타입 분리
- 기본 정보 직접 수정
- 락커 만료일
- PT/GX/필라테스 이력 세분화

이 항목들은 `기획 포함`이지만 `현재 구현은 부분구현`으로 정리했다.

## 4. 이전 문제와 현재 정리 방식

이전에는 아래 문제가 있었다.

- 화면설계서가 구현 기준 문서로 바뀌어 있었다.
- IA와 기능명세서는 기획 기준 문서였다.
- 문서 우선순위상 화면설계서가 최상위라서 체크리스트 항목이 빠진 것처럼 읽혔다.

현재는 아래 방식으로 정리했다.

- 화면설계서를 다시 `기획 기준 + 구현 상태 병기` 문서로 재정의
- IA, 기능명세서, 화면설계서 모두 같은 화면/기능 구조로 맞춤
- 체크리스트의 해석 차이는 `APP-기획연계정의서.md`에서 명시

## 5. 실제 차이점 요약

### 회원가입

- 체크리스트는 앱 내 직접 가입에 가깝게 읽힐 수 있다.
- 현재 기획은 `센터 CRM 등록 후 앱에서 휴대폰 인증으로 연동 완료`로 정리했다.

### 포인트/배지

- 체크리스트는 `포인트/배지` 표현을 사용한다.
- 현재 기획은 `마일리지/배지`로 정리했다.
- 즉, 포인트를 없앤 게 아니라 `마일리지`로 통일했다.

### 결제

- 체크리스트는 결제수단 선택과 영수증까지 요구한다.
- 현재 기획은 이를 `상품 스토어 + 개인 결제 + 결제 + 영수증` 구조로 확장해 포함한다.
- 실제 PG는 없지만 퍼블리싱 기준 사용 흐름은 현재 프로젝트에 반영했다.

### 개인정보·로그아웃

- 체크리스트는 개인정보와 로그아웃을 하나의 항목으로 본다.
- 현재 기획은 이를 `설정`, `약관/정책`, `동의관리`, `회원 탈퇴`, `로그아웃` 묶음으로 해석한다.
- 탈퇴는 즉시 계정 삭제가 아니라 `탈퇴 요청 접수` 형태로 정의했다.
