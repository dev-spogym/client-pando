#### 5. Supabase 연동 없이도 퍼블리싱 가능하도록 정리

- 실제 운영 DB가 연결되지 않아도 화면을 확인할 수 있도록 preview/mock 기준을 추가했습니다.
- 즉, 디자인 카탈로그가 아니라 `실제 화면 컴포넌트`를 직접 재사용하는 퍼블리싱 구조로 변경했습니다.

### 4-6. 퍼블리싱 구조 정리

- 퍼블리싱 경로: `/publishing/member`
- 현재 퍼블리싱 장수: `61장`
- 포함 범위:
  - 기본 화면
  - 탭 상태
  - 모달/바텀시트 상태
  - 완료 상태
  - 에러 상태
- 중요한 변경점:
  - 기존에는 퍼블리싱 전용 복제 UI였음
  - 현재는 실제 앱 라우트를 `preview=1` 모드로 직접 호출하는 구조로 변경
  - 따라서 퍼블리싱 화면과 실제 사용 화면의 차이를 줄였음

### 4-7. 현재 구현 상태 요약

- 체크리스트 기준 상태:
  - `구현 33건`
  - `부분구현 4건`
  - `미구현 0건`
- `부분구현`으로 남긴 항목:
  - OT1 / OT2 예약 타입의 실운영 분리
  - 기본 정보 직접 수정 기능
  - 락커 만료일 실데이터 반영
  - PT / GX / 필라테스 이력의 세부 분리 요약

### 4-8. 현재 문서/화면 반영의 의미

- 체크리스트 항목을 단순히 별도 메모로 관리하는 것이 아니라 실제 기획 문서와 화면 구조 안으로 다시 넣었습니다.
- 특히 다음 기준을 맞추는 데 집중했습니다.
  - 체크리스트에 있는 항목은 기획 문서 어디엔가 반영되어야 함
  - 기획 문서에 있는 항목은 실제 화면/퍼블리싱에서 확인 가능해야 함
  - 퍼블리싱은 실제 화면을 재사용해야 함

## 6. 대표님/클라이언트 보고용 요약

- 이번 주 `admin-pando`의 핵심은 문서량 증가가 아니라 문서를 운영 가능한 체계와 납품 가능한 품질로 보완한 것입니다.

  - 일반 이용권/개인 결제
  - 마일리지/배지
  - 온보딩/알림센터/대기예약/수업후기
  - 약관/동의관리/회원탈퇴
  - 실제 화면 재사용 기반 퍼블리싱 갤러리
- 향후 다음 단계는 `부분구현 4건`에 대한 운영 데이터 연결 및 세부 기능 마감입니다.
