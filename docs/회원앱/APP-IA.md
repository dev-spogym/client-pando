## 1. 핵심 원칙

- 회원앱은 `회원 중심 통합 앱`이다.
- 회원가입은 `CRM 등록 + 모바일 앱 휴대폰 인증 연동` 방식으로 해석한다.
- 체크리스트의 `포인트`는 `마일리지`로 통일한다.
- `배지`, `알림센터`, `온보딩`, `강사 상세`, `대기 예약`, `수업 후기`, `재등록 추천`을 회원앱 현재 기획 범위에 포함한다.
- PT 플랜 외에도 `헬스장 이용권`, `골프장 이용권`, `골프 레슨권`, `개인 결제`를 동일 결제 도메인 안에 포함한다.
- 실제 Supabase/PG 연동이 없어도 회원이 흐름을 이해할 수 있도록 `퍼블리싱 화면`과 `모의 사용 흐름`을 제공한다.
- 화면설계서/기능명세서/IA는 동일한 화면 ID와 기능 ID를 사용한다.

## 2. 역할 구조

| Role | 설명 |
|---|---|
| member | 일반 회원 |
| trainer | 트레이너 |
| golf_trainer | 골프강사 |
| fc | FC |
| staff | 스태프 |

회원앱 체크리스트 반영 작업의 중심은 `member` 영역이다.  
트레이너/골프강사/FC/스태프는 기존 IA를 유지하되 회원 흐름과 연결되는 화면만 참조한다.

## 5. 회원 탭 구조

```text
[ 홈 ] [ 예약 ] [ QR ] [ 이용권 ] [ MY ]
 MA-100  MA-120  MA-110  MA-131   MA-130
```

보조 화면은 탭 내부 스택으로 진입한다.

- 홈 하위: 리워드, 온보딩, 알림센터, 공지
- 예약 하위: 수업 상세, 골프 예약, 강사 상세, 대기 예약, 후기
- 이용권 하위: 상품 스토어, 상품 상세, 결제, 재등록 추천, 결제 내역, 영수증
- MY 하위: 설정, 약관, 동의관리, 탈퇴, 리워드, 알림, 공지, 온보딩

## 6. 주요 회원 플로우

### 6.1 CRM 회원 앱 연동 플로우

```text
CRM 회원 등록
  → 앱 설치
  → MA-002 앱 가입 / 연동
    → 이름/연락처 확인
    → 휴대폰 인증
    → 비밀번호 설정
  → MA-001 로그인
  → MA-127 온보딩 설문
  → MA-128 첫 루틴 제안
  → MA-100 홈
```

### 6.2 예약 / 대기 / 후기 플로우

```text
MA-100 또는 MA-120
  → MA-121 수업 상세
    → 예약 확정
    → 정원 마감 시 MA-124 대기 예약
    → 강사 보기 시 MA-125
  → MA-122 내 예약 / 수업 이력
    → 완료 수업 선택
    → MA-126 수업 후기
```

### 6.3 Golf 강사 예약 / 타석 예약 플로우

```text
MA-100 또는 MA-123
  → 골프 강사 예약 탭
    → 강사/시간대 선택
    → 예약 확인
    → 내 골프 레슨 예약 확인
  → 골프 타석 예약 탭
    → 타석 선택
    → 예약 확인
    → 만석 시 대기 등록
```

### 6.4 상품 구매 / 개인 결제 플로우

```text
MA-131 또는 MA-134 또는 MA-141
  → MA-134 상품 스토어
    → MA-139 상품 상세
    → MA-140 결제하기
      → 카드 / 계좌이체 / 네이버페이 / 카카오페이 선택
      → 마일리지 사용
      → MA-142 영수증 상세
  → MA-141 개인 결제
    → 수동 결제 항목 생성
    → MA-140 결제하기
```

### 6.5 리워드 / 재등록 플로우

```text
MA-100 홈
  → MA-136 리워드 센터
    → 쿠폰 / 마일리지 / 배지 확인
  → MA-138 재등록 추천 플랜
    → 상품 결제 또는 결제 이력 확인
```

### 6.6 약관 / 동의 / 탈퇴 플로우

```text
MA-155 설정
  → MA-156 약관 / 정책
  → MA-157 동의관리
    → 마케팅 수신 상태 변경
  → MA-158 회원 탈퇴
    → 사유 선택
    → 추가 의견 입력
    → 탈퇴 요청 저장
```

## 7. 체크리스트 반영 메모

- `포인트`는 `마일리지`로 통일한다.
- `배지`는 `리워드 센터` 내 별도 컬렉션으로 추가한다.
- `온보딩 설문`과 `첫 루틴 제안`은 `MA-127~128`로 독립시킨다.
- `회원가입`은 공개 회원 생성이 아니라 `CRM 등록 회원의 앱 연동`으로 정의한다.
- `결제수단 선택·결제`, `영수증`, `개인 결제`, `헬스장/골프장 이용권 구매`는 `MA-133~142` 영역으로 확장한다.
- `개인정보·로그아웃`은 설정 단일 화면이 아니라 `설정 + 약관/정책 + 동의관리 + 탈퇴` 묶음으로 해석한다.
