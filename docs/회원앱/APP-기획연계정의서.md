## 3. 이번 정리의 핵심 원칙

### 3.1 회원가입

- 회원 기본 생성 주체는 `CRM`이다.
- 앱에서는 `휴대폰 인증 + 비밀번호 설정`으로 연동/가입을 완료한다.
- 즉, 체크리스트의 `회원가입`은 현재 기준에서 `CRM 선등록 + 모바일 앱 연동`으로 해석한다.
- 공개형 즉시 회원 생성은 현재 기본 정책이 아니다.

### 3.2 리워드

- 체크리스트의 `포인트`는 현재 프로젝트/기획에서 `마일리지`로 통일한다.
- `포인트 사용`은 `마일리지 사용`으로 해석한다.
- `배지`는 후속이 아니라 현재 회원앱 기획 범위에 포함한다.
- 따라서 리워드 체계는 `쿠폰 + 마일리지 + 배지` 3축으로 정의한다.

### 3.3 온보딩

- 체크리스트의 `운동 목적/성향`, `통증/체형 정보`, `첫 루틴 제안`은 현재 회원앱 범위에 포함한다.
- 앱 연동 직후 또는 첫 로그인 직후 회원이 온보딩을 진행한다.
- FMS/체형 리포트는 온보딩 입력값과 측정 기록을 함께 참조한다.

### 3.4 현재 프로젝트 유지 원칙

- 기존 라우트 구조와 회원 탭 구조는 유지한다.
- 없는 백엔드 원장을 한 번에 새로 만들지 않는다.
- 현재 프로젝트에서 바로 흡수 가능한 기능은 `UI/로컬 상태/Supabase 조회` 조합으로 추가한다.
- 기능명세서와 화면설계서에는 `기획 기준`과 `현재 구현 상태`를 동시에 남긴다.

### 3.5 결제 / 상품 확장

- 체크리스트의 결제 범위는 `PT 플랜`에만 한정하지 않는다.
- 헬스장 이용권, 골프장 이용권, 골프 레슨권, 재등록 상품, 개인 결제를 모두 회원앱 결제 범위에 포함한다.
- 실제 PG가 없어도 `상품 스토어`, `상품 상세`, `결제`, `영수증`, `개인 결제` 화면은 퍼블리싱 기준으로 제공한다.

### 3.6 개인정보 / 동의 / 탈퇴

- 체크리스트의 `개인정보·로그아웃`은 설정 한 화면이 아니라 `약관/정책`, `동의관리`, `회원 탈퇴`, `로그아웃`까지 포함한 묶음으로 해석한다.
- 회원 탈퇴는 현재 단계에서 `탈퇴 요청 접수` 화면으로 구현한다.

## 4. 체크리스트 용어 매핑

| 체크리스트 용어 | 회원앱 적용 용어 | 처리 기준 |
|---|---|---|
| 포인트 | 마일리지 | 동일 개념으로 통합 |
| 포인트 사용 | 마일리지 사용 | 결제 리워드 차감 의미 |
| 배지 | 배지 | 신규 포함 |
| 회원가입 | CRM 등록 + 앱 연동 | 휴대폰 인증 기반 |
| 온보딩 설문 | 운동 목적/성향 + 통증/체형 입력 | 신규 포함 |
| 첫 루틴 제안 | 온보딩 완료 결과 | 신규 포함 |
| 공지/이벤트 배너 | 홈 공지/이벤트 카드 | 공지 중심 + 이벤트 표현 확장 |
| FMS/체형 리포트 | 체성분/FMS 통합 화면 | 동일 도메인으로 묶음 |

## 5. 체크리스트 반영 방향

| 영역 | 기존 상태 | 이번 기준 |
|---|---|---|
| 홈 | 공지/출석/오늘 수업 중심 | PT/GX/Golf/리워드 진입과 마일리지/배지 요약 포함 |
| 예약 | 수업 목록/상세 중심 | 대기, 강사 상세, 후기까지 회원 흐름 완성 |
| 골프 | 타석 예약 중심 | 골프 강사 예약 + 타석 예약 + 이용권 구매 연계 |
| 결제 | 재등록/결제 이력 중심 | 상품 스토어, 개인 결제, 결제, 영수증까지 확장 |
| 리워드 | 쿠폰/마일리지 중심 | 배지 포함 리워드 센터로 확장 |
| 마이페이지 | 조회 중심 | 알림센터, 설정, 약관/동의/탈퇴, 온보딩, 재등록 진입 포함 |
| 알림 | 문서상만 존재 | 회원 기준 실사용 화면으로 포함 |
| 온보딩 | 문서 누락 | 현재 범위 포함 |
| 재등록 | FC 중심 | 회원용 추천 플랜 화면 포함 |

## 6. 구현 상태 표기 규칙

화면설계서와 기능명세서에는 아래 상태를 함께 적는다.

- `구현`: 현재 프로젝트에서 바로 사용 가능
- `보강구현`: 이번 정리에서 화면/흐름을 추가함
- `부분구현`: 일부만 동작하며 운영 연동이 더 필요함
- `후속`: 기획에는 포함하되 현재 프로젝트에는 미반영

## 7. 이번 정리 결과

- 체크리스트 37개 항목은 회원앱 IA, 화면설계서, 기능명세서에서 모두 추적 가능해야 한다.
- `포인트`는 `마일리지`로 통일한다.
- `배지`, `알림센터`, `온보딩`, `대기예약`, `강사상세`, `수업후기`, `재등록 추천`은 회원앱 현재 기획 범위에 편입한다.
- 회원가입은 `CRM 등록 + 모바일 앱 연동`으로 정리한다.
- `골프 강사 예약`, `상품 스토어`, `개인 결제`, `결제`, `영수증`, `약관/동의/탈퇴`도 같은 기준으로 편입한다.
