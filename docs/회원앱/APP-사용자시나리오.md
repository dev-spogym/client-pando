### 1.2 핵심 유저 저니 (End-to-End, 화면 ID 포함)

---

#### 저니 1: 신규 회원 첫 앱 사용

```
[오프라인] 센터 방문 → 상담 → 이용권 구매 → CRM에 회원 등록 (이름, 연락처, role=member)
    ↓
[앱 설치] 스태프가 "FitGenie 앱 설치해주세요" 안내
    ↓
MA-004 스플래시 → 온보딩 가이드 (최초 1회)
    ↓
MA-001 로그인 화면 → "앱 연동/가입" 링크 탭
    ↓
MA-002 앱 연동 Step 1: 이름 + 연락처 입력
    → CRM members 테이블에서 일치하는 회원 조회
    → 일치하면 SMS 인증번호 발송
    ↓
MA-002 Step 2: SMS 인증번호 입력 (6자리, 3분 유효)
    → 재발송 60초 쿨다운
    → 5회 오류 시 세션 무효화
    ↓
MA-002 Step 3: 비밀번호 설정 (8자 이상, 영문+숫자 조합)
    → 비밀번호 확인 입력
    → app_linked_at 업데이트
    ↓
MA-001 로그인 → 연락처 + 비밀번호 입력
    → JWT Access Token(1시간) + Refresh Token(30일) 발급
    → role=member 확인 → MemberTabNavigator 진입
    ↓
MA-100 회원 홈 진입
    → 이용권 D-day 카드 확인
    → 오늘 예약 현황
    → 출석 현황
    → 공지사항 미리보기
    ↓
MA-110 QR 체크인으로 첫 출석 → 진동 + 성공 애니메이션 → 🎉
```

#### 저니 2: 수업 예약 → 출석 → 완료

```
MA-100 홈 → 하단 탭 "예약" 탭
    ↓
MA-120 수업 목록
    → 날짜 선택 (주간 캘린더 스크롤)
    → PT/GX 탭 필터
    → 트레이너별 필터
    → 수업 선택
    ↓
MA-121 수업 상세
    → 수업명, 시간, 트레이너, 룸, 잔여석 ("잔여 3/10석")
    → [예약하기] 버튼
    ↓
예약 확인 모달
    → "18:00 PT 수업을 예약하시겠습니까?"
    → [확인] → "예약이 완료되었습니다" 토스트
    → 이용권 잔여 횟수 차감 표시
    ↓
(수업 1시간 전) 푸시 알림: "14:00 PT 수업이 있습니다"
    ↓
헬스장 도착 → MA-110 QR 체크인 → 출석 완료
    ↓
(수업 시간) 트레이너 MA-212에서 [수업 시작] 탭
    → 수업 상태: scheduled → in_progress
    → 회원 예약 상태: "진행중"으로 변경
    ↓
(수업 종료) 트레이너 MA-212에서:
    1. 출석 체크 → 참석 선택
    2. [수업 완료] 버튼
    3. 서명패드에 서명
    4. 수업 상태: in_progress → completed
    ↓
회원 MA-122에서 예약 상태: "완료" 확인
```

#### 저니 3: 골프 쌍방서명 -- 대면 모드

```
(레슨 종료, 회원이 옆에 있음)
    ↓
강사 MA-212 수업 상세 → [수업 완료] 버튼
    ↓
MA-312 Step 1: 수업 정보 확인
    → 날짜: 2026-04-17
    → 시간: 09:30 ~ 10:20
    → 회원: 김철수
    → 내용: 드라이버 스윙 교정 (어드레스 자세, 백스윙 궤도)
    ↓
MA-312 Step 2: 강사 서명 패드
    → 강사가 서명 → [서명 완료]
    → 서명 이미지 + 타임스탬프 + 디바이스 정보 저장
    ↓
MA-312 Step 3: 서명 방식 선택
    → [대면 서명] 버튼 탭
    ↓
MA-312 Step 3-A: 회원 서명 화면 (디바이스를 회원에게 전달)
    → 상단 안내: "수업을 정상 수강하였음을 확인합니다"
    → 회원이 서명 → [서명 완료]
    → 서명 이미지 + 타임스탬프 + 디바이스 정보 저장
    ↓
MA-312 Step 4: 양측 서명 완료 화면
    → 강사 서명 이미지 + 서명 시각
    → 회원 서명 이미지 + 서명 시각
    → 수업 상태: completed ✅
    → [확인서 보기] → MA-313 레슨 확인서
    → PDF 저장 / 공유 가능
```

#### 저니 4: 골프 쌍방서명 -- 원격 모드 (회원이 먼저 떠남)

```
(레슨 종료, 회원 퇴장)
    ↓
강사 MA-312 → Step 1~2 동일 (수업 정보 확인 + 강사 서명)
    ↓
MA-312 Step 3: 서명 방식 선택 → [원격 서명 요청] 탭
    ↓
시스템 처리:
    → 회원 앱으로 푸시 알림 발송 (딥링크 포함)
    → 푸시 내용: "이준호 프로의 레슨 완료 서명을 요청합니다"
    ↓
강사 MA-312 대기 화면:
    → "회원 서명 대기 중..."
    → "김철수님에게 알림을 발송했습니다"
    → 강사는 이 화면을 닫고 다른 작업 가능
    ↓
(회원 측) 푸시 알림 수신 → 탭 → 서명 화면 진입
    → "수업을 정상 수강하였음을 확인합니다" 확인
    → 서명 → [서명 완료]
    ↓
(강사 측) 실시간 갱신 또는 푸시: "김철수님이 서명을 완료했습니다 ✅"
    → 레슨 확인서 자동 생성 → MA-313에서 확인 가능
    ↓
타임아웃 시나리오:
    → 12시간 경과, 미서명 → 시스템이 푸시 알림 자동 재발송
    → 24시간 경과, 미서명 → 서명 상태: expired ⏰
    → 강사에게 "서명 만료" 알림 → 수업 상세에서 재요청 가능
```

#### 저니 5: 예약 취소 (마감 전/후 분기)

```

    ↓
[분기 A] 마감 전 (수업까지 2시간 이상 남음):
```

## Part 2. 비즈니스 정책 상세

### 2.1 출석 정책

| 정책 | 내용 | 배경 |
|------|------|------|
| QR 체크인 기본 | 회원이 직접 QR 스캔으로 출석 | 부정 출석 방지, 자동화. 대리 출석 원천 차단. |
| 수동 출석 | 스태프/트레이너만 가능, 사유 입력 필수 | QR 불가 시 대안. 사유 기록으로 남용 추적. |
| 중복 체크인 방지 | 당일 동일 유형 1회만 | 출석 통계 왜곡 방지. 잘못된 KPI 산출 방지. |
| 이용권 없어도 체크인 | 가능하되 경고 표시 | 체험 방문/재등록 상담 대응. 만료 후 방문 회원 데이터 수집. |
| 체크인 가능 시간 | 센터 운영 시간 내 | 비정상 시간 출석 방지. 새벽/심야 허위 출석 차단. |
| QR 코드 유효 시간 | 1분 (매번 갱신) | 스크린샷 캡처 후 타인 사용 방지. |

**센터별 설정 가능 항목:**
- 체크인 가능 시간 (시작~종료)
- 중복 체크인 허용 여부 (일부 24시간 센터는 오전/오후 분리 체크인)
- QR 코드 갱신 주기 (30초~3분)

### 2.2 예약 정책

| 정책 | 내용 | 배경 |
|------|------|------|
| 취소 마감 | 수업 시작 N시간 전 (기본 2시간) | 다른 회원 예약 기회 보호. 빈자리 재판매 시간 확보. |
| 마감 후 취소 | 노쇼와 동일 처리 (횟수 차감, 노쇼 카운트) | 페널티 없으면 "일단 예약하고 안 가기" 남발. |
| 동시 예약 제한 | 같은 시간대 1개만 | 중복 예약 방지. 자원 낭비 방지. |
| 최대 사전 예약 | N일 전까지 (기본 7일) | 너무 먼 미래 예약 방지. 노쇼 확률이 기간에 비례하여 증가. |
| 정원 관리 | 정원/타석 초과 시 직접 예약 불가, 대기 가능 시 대기 등록 노출 | 수업 품질 유지. 안전 기준 준수. 병렬 자원(골프 타석) 관리 필요. |
| 대기 등록 | 센터 설정에 따라 활성/비활성 | 센터마다 운영 방식이 다름. 소규모 센터는 불필요. |
| PT 예약 제한 | 담당 트레이너 수업만 예약 가능 | PT는 1:1 관계. 타 트레이너 수업 예약 방지. |
| 골프 타석 선택 | 골프 타입 수업은 예약 전 사용 가능한 타석/부스를 선택 | 골프는 같은 시간에 여러 타석이 병렬로 열리므로 자원 단위 선택이 필요. |
| OT1/OT2 분기 | OT1은 최초 진단/상담, OT2는 루틴 확정 후 후속 세션으로 구분 | 온보딩 품질 확보. 일반 PT와 목적/준비물이 다름. |
| 후기 요청 | 완료 수업에 한해 24시간 내 1회 만족도 평가 가능 | 서비스 품질 측정, 강사 피드백, 프로그램 개선 근거 확보. |

**센터별 설정 가능 항목:**
- 취소 마감 시간 (1~24시간)
- 최대 사전 예약일 (3~30일)
- 대기 등록 여부
- 자동 확정 여부 (대기 1순위 즉시 확정 / 수동 확인)
- 동시 예약 최대 건수 (1~5건)
- 골프 타석형 수업 여부

**왜 취소 마감 시간이 2시간인가?**
- 인기 수업은 항상 대기 인원이 있음
- 2시간이면 대기 회원이 알림 받고 준비할 시간 충분
- 1시간은 너무 촉박, 4시간은 회원 불편 → 2시간이 업계 표준

### 2.3 수업 서명 정책

| 항목 | 일반 PT/GX | 골프 레슨 |
|------|-----------|----------|
| 서명 주체 | 강사만 (단방향) | 강사 + 회원 (쌍방) |
| 필수 여부 | 필수 | 필수 |
| 서명 방식 | 앱 내 SignaturePad | 앱 내 SignaturePad |
| 회원 서명 | 불필요 | 대면 또는 원격 선택 |
| 원격 서명 대기 시간 | - | 24시간 |
| 알림 재발송 | - | 12시간 후 자동 1회 |
| 서명 만료 | - | 24시간 후 자동 만료 |
| 만료 후 조치 | - | 강사가 수업 상세에서 재요청 가능 |
| 증빙 산출물 | 서명 이미지 | 서명 이미지 + PDF 레슨 확인서 |
| 저장 데이터 | 서명 이미지, 타임스탬프 | 서명 이미지, 타임스탬프, 디바이스 정보, GPS(선택) |

**왜 골프만 쌍방서명인가?**
- 골프 레슨 단가: 1회 10~20만원 (PT 3~5만원의 3~5배)
- 실제 분쟁 사례: "수업을 안 했는데 수업료가 빠졌다" 민원이 골프에서 빈번
- 쌍방서명 = 양측이 "수업이 정상 진행됐다"는 법적 증거
- 서명 이미지 + 타임스탬프 + 디바이스 정보 = 위변조 방지
- PT는 단가가 낮고, 트레이너-회원 간 신뢰 관계가 강해 단방향 서명으로 충분

**서명 상태 전이도:**

```
(수업 시작)
    → in_progress
        → (강사가 수업 완료 요청)
            → pending_instructor_sign
                → (강사 서명 완료)
                    → pending_member_sign
                        ├── (회원 서명 완료) → completed ✅
                        ├── (회원 서명 거부) → declined ❌ → 매니저 에스컬레이션
                        ├── (12시간 경과) → 알림 자동 재발송 (상태 유지)
                        └── (24시간 경과) → expired ⏰ → 강사 재요청 가능
```

### 2.4 노쇼 정책

| 누적 횟수 | 조치 | 복원 조건 |
|----------|------|----------|
| 1회 | 경고 푸시 알림 | 자동 초기화 없음 |
| 2회 | 경고 + 담당 FC/트레이너에게 알림 | 자동 초기화 없음 |
| 3회 | 예약 기능 제한 (기본 1주) | 제한 기간 경과 후 자동 해제 |
| 5회 | 예약 기능 제한 (기본 2주) | 제한 기간 경과 후 자동 해제 |
| 매니저 면제 | 노쇼 카운트 1건 차감 | CRM에서 센터장/매니저만 처리 가능. 비상 상황(입원, 사고 등) 증빙 필요. |

**센터별 설정 가능 항목:**
- 제한 시작 횟수 (2~5회)
- 제한 기간 (3일~1개월)
- 노쇼 카운트 초기화 주기 (월초/분기초/수동)
- 면제 정책 활성화 여부

**왜 노쇼 페널티가 필요한가?**
- 노쇼 1건 = 다른 회원 1명이 수업을 못 들음
- 인기 수업은 항상 정원 마감 → 노쇼 시 빈자리 = 수익 손실
- 페널티 없으면 "일단 예약하고 안 가기" 남발
- 업계 데이터: 노쇼 페널티 도입 시 노쇼율 평균 40% 감소

**노쇼 카운트 방식:**
- 마감 후 취소 = 노쇼 1회로 카운트
- 트레이너가 수업에서 노쇼 처리 = 노쇼 1회로 카운트
- 예약 없이 안 온 경우 = 카운트하지 않음 (예약 기반 추적)

### 2.8 데이터 접근 정책

| 역할 | 회원 데이터 | 수업 데이터 | 상담 데이터 | 결제 데이터 | 체성분 데이터 |
|------|-----------|-----------|-----------|-----------|-------------|
| member | 본인만 (CRUD) | 본인 예약만 (R) | - | 본인만 (R) | 본인만 (R) |
| trainer | 담당 회원 (R) | 본인 수업 (CRUD) | - | - | 담당 회원 (CRU) |
| golf_trainer | 담당 회원 (R) | 본인 수업 + 서명 (CRUD) | - | - | 담당 회원 (CRU) |
| fc | 담당 회원 (R) | - | 본인 상담 (CRUD) | - | - |
| staff | 전체 (R) | 전체 (R) | - | - | - |

> R=읽기, C=생성, U=수정, D=삭제(soft)

**왜 이렇게 나누는가?**
- **최소 권한 원칙 (Principle of Least Privilege)**: 업무에 필요한 데이터만 접근. 트레이너가 결제 정보를 볼 이유 없음.
- **개인정보 보호**: 회원 연락처, 결제 정보는 민감 데이터. GDPR/개인정보보호법 준수.
- **책임 소재 명확**: 누가 어떤 데이터를 봤는지 추적 가능. 감사 로그 기록.

**API 접근 제어 구현:**
- Supabase RLS (Row Level Security) 정책으로 DB 레벨에서 강제
- API 미들웨어에서 role 기반 라우트 가드
- 스태프의 회원 화면은 수정 API 호출 자체를 차단 (읽기전용 UI + API 권한 분리)

### 회원 ↔ 트레이너

## Part 4. 용어 정의

| 용어 | 영문 | 정의 |
|------|------|------|
| 이용권 | Ticket | 회원이 구매한 헬스장/골프장 이용 권리. 기간제, 횟수제, 기간+횟수제 3가지 유형이 있다. |
| 홀딩 | Holding | 이용권 일시 정지. 기간제 이용권만 가능하며, 홀딩 기간만큼 종료일이 연장된다. 7~30일 범위. |
| 노쇼 | No-show | 예약 후 무단 불참. 마감 시간 이후 취소도 노쇼로 간주한다. 누적 시 예약 제한 페널티 적용. |
| 쌍방서명 | Dual Signature | 골프 레슨 시 강사+회원 양측이 서명하여 수업 완료를 확인하는 방식. 법적 증빙 목적. |
| 레슨 확인서 | Lesson Certificate | 쌍방서명 완료 후 자동 생성되는 수업 증빙 문서. PDF 다운로드 및 공유 가능. 정산 첨부 자료. |
| FC | Fitness Consultant | 회원 영입, 상담, 재등록을 담당하는 영업 직원. 센터 매출의 최전선. |
| OT | Orientation | 신규 회원 대상 센터 안내 및 기구 사용법 교육. FC가 진행하며 상담 유형 중 하나로 기록. |
| 리드 | Lead | 아직 등록하지 않은 잠재 고객. 상담 대상자. 이름+연락처만 임시 등록된 상태. |
| 완강 | Course Completion | 횟수제 이용권의 전체 횟수를 모두 사용한 상태. 잔여 0회. |
| 체성분 | Body Composition | 체중, 골격근량, 체지방률, BMI 등 신체 측정 데이터. InBody 등 기기로 측정. |
| 딥링크 | Deep Link | 푸시 알림 탭 시 앱의 특정 화면으로 바로 이동시키는 URL 스킴. |
| 낙관적 잠금 | Optimistic Locking | 동시 요청 충돌 해결 방식. 먼저 커밋한 트랜잭션이 성공, 나머지는 충돌 에러 반환. |
| RLS | Row Level Security | Supabase/PostgreSQL의 행 단위 접근 제어. role 기반으로 조회 가능한 데이터를 DB 레벨에서 제한. |
| PG | Payment Gateway | 결제 대행 서비스. 본 앱은 토스페이먼츠를 사용. |
| soft delete | Soft Delete | 물리 삭제 대신 deleted_at 타임스탬프를 기록하여 논리적으로 삭제 처리. 이력 추적 목적. |
