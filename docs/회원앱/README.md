## 1. 프로젝트 배경과 문제 정의

### 1.1 현재 상황 — 앱 없이 발생하는 문제들

헬스장과 골프 연습장은 여전히 전화, 종이, 엑셀, 개인 메신저에 의존해 운영된다. 역할별로 겪는 불편은 다음과 같다.

| 역할 | 현재 불편 | 구체적 상황 |
|------|----------|------------|
| **회원** | 전화로 예약, 이용권 잔여일 모름, 출석 확인 불가, 상품 구매하려면 방문 필요 | "PT 남은 횟수가 몇 회인지 매번 트레이너에게 물어봐야 한다", "예약 변경하려면 영업시간에 전화해야 한다" |
| **트레이너** | 수업 스케줄을 종이/엑셀로 관리, 회원 출석 체크를 수기로, 체성분 기록 분실 | "엑셀에 스케줄 적다가 실수로 덮어씌워서 이번 주 일정이 날아갔다", "회원 인바디 기록을 종이에 적어뒀는데 어디 갔는지 모르겠다" |
| **골프강사** | 레슨 완료 증빙 없어 수업료 분쟁 발생, 서면 서명 관리 어려움 | "회원이 수업을 안 했다고 주장하는데 증거가 없다", "종이 서명지를 분실해서 정산 때 문제가 됐다" |
| **FC (상담사)** | 상담 이력을 개인 메모/카톡에 기록, 만료 회원 놓침, 전환율 추적 불가 | "상담 내용을 카톡 메모에 적어뒀다가 폰 교체하면서 전부 유실됐다", "만료된 회원에게 연락하는 걸 깜빡해서 이탈했다" |
| **스태프** | 회원 확인에 시간 소요, 출석 수기 기록 오류 | "회원 얼굴은 아는데 이름이 기억 안 나서 CRM을 뒤져야 한다", "수기 출석 체크에서 날짜를 잘못 적었다" |
| **센터장** | 현장 상황을 실시간으로 파악 불가, CRM은 PC에서만 접근 가능 | "외출 중에 오늘 출석률이나 수업 현황을 확인할 수 없다", "회원 문의 전화가 오면 사무실에 돌아가서 PC를 켜야 한다" |

### 1.2 앱으로 해결되는 것 — 1:1 대응

| 현재 문제 | 앱의 해결 방식 |
|----------|---------------|
| 전화로 예약 | 앱에서 수업 시간표 확인 → 원터치 예약/취소 |
| 이용권 잔여일 모름 | 홈 화면에 D-day 카운트 + 잔여 횟수 상시 표시 |
| 출석 확인 불가 | QR 체크인으로 자동 출석 기록 + 이력 조회 |
| 상품 구매 시 방문 필요 | 앱 내 온라인 스토어에서 이용권/상품 결제 |
| 수업 스케줄 종이/엑셀 관리 | 앱 캘린더에서 수업 일정 자동 동기화 |
| 출석 체크 수기 기록 | 수업 시작 버튼 → 자동 출석 + 서명까지 원스톱 |
| 체성분 기록 분실 | 앱에서 즉시 기록 → 클라우드 영구 보관 + 추이 차트 |
| 레슨 완료 증빙 없음 | 쌍방서명(강사+회원) + 타임스탬프 → 법적 증거력 확보 |
| 서면 서명 관리 어려움 | 전자 서명 + 레슨 확인서 PDF 자동 생성 |
| 상담 이력 유실 | 앱에서 상담 결과 즉시 기록 → CRM과 실시간 동기화 |
| 만료 회원 놓침 | D-30/D-14/D-7/D-3/D-1 자동 알림 → FC에게 푸시 |
| 전환율 추적 불가 | FC KPI 대시보드에서 상담 건수, 등록 전환율 자동 집계 |
| 회원 확인에 시간 소요 | 앱에서 이름/연락처 검색 → 즉시 회원 상태 확인 |
| 현장 상황 실시간 파악 불가 | 모바일 앱으로 어디서든 출석 현황, 수업 상태, KPI 확인 |

### 1.3 목적

FitGenie 회원앱은 CRM 백오피스와 연동되는 **역할 기반 통합 모바일 앱**이다.

- **일반 회원**: 출석 체크인, 수업 예약, 이용권 조회, 온라인 구매 등 헬스장 이용 전반을 앱에서 처리한다.
- **트레이너 / 골프강사**: 수업 캘린더, 수업 시작/완료, 담당 회원 관리, 체성분/신체정보 기록 등 현장 업무를 지원한다.
- **FC (Fitness Consultant)**: 리드 관리, 상담 이력 등록/조회, 만료 예정 회원 재등록 상담 등 영업 활동을 지원한다.
- **스태프**: 회원 조회(읽기 전용), 수동 출석 처리 등 프론트 데스크 업무를 지원한다.

한 계정은 하나의 역할만 가지며(멀티 역할 없음), 역할은 CRM에서 사전 지정된다. 로그인 후 role 값에 따라 앱의 네비게이션과 화면이 자동으로 분기된다.

앱의 로그인 진입은 `회원 로그인`과 `직원 로그인` 두 갈래로 구성한다.

- `회원 로그인`: 전화번호 + 비밀번호
- `직원 로그인`: 아이디 + 비밀번호
- 직원 로그인 하위 선택: `트레이너`, `FC`, `스태프`
- `golf_trainer` 계정은 직원 로그인 화면에서는 `트레이너` 계열로 묶어 진입한다.
- 선택한 직원 유형과 실제 계정 role이 다르면 로그인되지 않는다.

### 1.4 CRM 연동

| 항목 | 내용 |
|------|------|
| DB | Supabase DB 공유 (CRM과 동일 인스턴스) |
| 접근 제어 | RLS(Row Level Security)로 역할별 데이터 접근 범위 제어 |
| API | CRM 확장 API + 앱 전용 엔드포인트 |
| 인증 | Supabase Auth — 회원(전화번호 + 비밀번호), 직원(아이디 + 비밀번호) + role 기반 화면 분기 |
| 실시간 | Supabase Realtime — 예약 현황, 알림 실시간 반영 |

### 1.5 역할 분기 구조

```
앱 실행
  └─ 로그인 화면 (MA-001)
       ├─ 회원 로그인
       │    └─ member       → MemberNavigator    (홈/예약/QR/이용권/MY)
       └─ 직원 로그인
            ├─ 트레이너 선택 → trainer, golf_trainer, admin → TrainerNavigator (홈/회원/일정/피드백/MY)
            ├─ FC 선택      → fc                         → FcNavigator       (홈/상담/회원/KPI/설정)
            └─ 스태프 선택   → staff                      → StaffNavigator    (홈/회원/출석/일정/설정)
```

## 2. 사용자 페르소나

### 2.1 김민수 (30세, 직장인 회원)

| 항목 | 내용 |
|------|------|
| **배경** | IT 회사 재직, 퇴근 후 헬스장 다니는 직장인. PT 10회 등록. |
| **앱 사용 패턴** | 출근길에 오늘 수업 확인 → 헬스장 도착하면 QR 체크인 → PT 후 체성분 변화 확인 |
| **Pain Point** | 예약 변경하려면 전화해야 했음. 이용권 남은 횟수를 몰라서 매번 물어봤음. 체성분 변화를 직접 비교할 수 없었음. |
| **핵심 기능** | QR 체크인, 수업 예약/취소, 이용권 조회, 체성분 차트 |
| **기대 효과** | "출근길 지하철에서 예약 확인하고, 도착하면 QR만 찍으면 끝. PT 끝나고 인바디 변화를 그래프로 보니까 동기부여 된다." |

### 2.2 박서연 (28세, PT 트레이너)

| 항목 | 내용 |
|------|------|
| **배경** | 하루 6~8세션 PT 진행, 담당 회원 15명. |
| **앱 사용 패턴** | 아침에 오늘 수업 확인 → 수업 시작 버튼 → 출석 체크 → 수업 끝나면 서명 받고 완료 → 체성분 기록 |
| **Pain Point** | 엑셀로 스케줄 관리하다 실수 발생. 수업 완료 서명을 종이로 받아서 분실. 회원 체성분 기록이 흩어져 있음. |
| **핵심 기능** | 수업 캘린더, 수업 시작/완료/서명, 체성분 기록, PT 횟수 관리 |
| **기대 효과** | "수업 시작 버튼 누르면 출석도 자동이고, 끝나면 서명 받으면서 체성분도 바로 기록. 종이 하나 안 쓴다." |

### 2.3 이준호 (42세, 골프 프로)

| 항목 | 내용 |
|------|------|
| **배경** | 실내 골프 연습장에서 1:1 레슨, 주 20세션 진행. |
| **앱 사용 패턴** | 수업 완료 후 "쌍방서명"으로 레슨 증빙 → 월말에 확인서 모아서 정산 |
| **Pain Point** | **골프 레슨은 1회 10~20만원으로 금액이 크다.** "수업을 제대로 했는지"에 대한 분쟁이 종종 발생. 종이 서명지 관리가 불편하고, 분실 시 증거가 없음. |
| **핵심 기능** | 쌍방서명 (강사+회원 모두 서명해야 수업 완료), 레슨 확인서 PDF |
| **기대 효과** | "수업 끝나고 서로 서명하면 자동으로 확인서가 만들어진다. 분쟁이 생겨도 날짜, 시간, 서명이 다 남아있으니 걱정 없다." |

### 2.4 최은영 (35세, FC/상담사)

| 항목 | 내용 |
|------|------|
| **배경** | 신규 회원 상담, 재등록 상담 담당. 하루 5~8건 상담 진행. |
| **앱 사용 패턴** | 아침에 오늘 상담 일정 확인 → 상담 후 바로 결과 기록 → 만료 예정 회원에게 전화 |
| **Pain Point** | 상담 내용을 카톡 메모에 적어뒀다가 유실. 만료 회원 추적을 수동으로. 전환율을 직접 계산해야 했음. |
| **핵심 기능** | 상담 이력 관리, 만료 예정 알림, FC KPI 대시보드 |
| **기대 효과** | "상담 끝나고 바로 결과를 기록하면 CRM에도 반영되고, 만료 7일 전 회원 목록이 자동으로 뜬다. 전환율도 알아서 계산된다." |

### 2.5 정하나 (24세, 프론트 스태프)

| 항목 | 내용 |
|------|------|
| **배경** | 프론트 데스크 근무, 회원 출입 관리. |
| **앱 사용 패턴** | 회원이 QR 안 될 때 수동 출석 → 회원 이용권 상태 확인 |
| **Pain Point** | CRM은 PC라서 자리 비울 때 확인 불가. 회원 이름이 기억 안 날 때 확인하려면 PC로 돌아가야 함. |
| **핵심 기능** | 회원 조회 (읽기전용), 수동 출석 |
| **기대 효과** | "PC 앞에 없어도 폰으로 바로 회원 검색해서 확인할 수 있다. QR 오류일 때 수동 출석도 바로 처리." |

## 3. 핵심 비즈니스 정책

> 각 정책에는 **"왜 이런 규칙이 있는지"** 이유를 함께 기술한다. 센터마다 운영 방식이 다르므로, 가능한 범위에서 센터별 설정을 지원한다.

### 3.1 출석 정책

| 규칙 | 내용 | 이유 |
|------|------|------|
| QR 체크인 기본 | 회원이 앱 QR을 스캔하여 출석 | 자동화된 출석 기록, 통계 정확도 확보 |
| 수동 출석은 스태프만 가능 | 회원 본인은 수동 출석 불가 | 부정 출석 방지 |
| 당일 중복 체크인 불가 | 같은 날 2회 이상 체크인 차단 | 출석 통계 왜곡 방지 |
| 이용권 없는 회원도 체크인 가능 | 체크인은 되지만 경고 표시 | 체험/방문 회원 대응, 프론트 스태프에게 알림 |

### 3.2 예약/취소 정책

| 규칙 | 내용 | 이유 |
|------|------|------|
| 취소 마감 시간 | 수업 시작 N시간 전까지 취소 가능 (N은 센터별 설정, 기본 2시간) | 다른 회원의 예약 기회 보호 |
| 마감 후 취소 | 취소 마감 이후 취소 시 노쇼와 동일 처리 | 무책임한 직전 취소 방지 |
| 대기 등록 | 정원 초과 시 대기 등록 가능 여부는 센터 설정 | 센터마다 운영 방식이 다름 |
| 동일 시간대 중복 예약 | 불가 | 한 회원이 같은 시간에 여러 수업 예약하는 것 방지 |
| 회원의 트레이너 예약 요청 | 회원이 강사 가능 시간을 보고 PT/골프 레슨을 요청할 수 있음 | 전화/메신저 예약 의존도 감소 |
| 회원 요청 예약 승인 | 회원이 직접 만든 트레이너 대상 예약은 강사 승인 후 확정 | 강사 일정 통제, 중복 방지 |
| 강사 직접 배정 | 강사가 자신의 캘린더에서 회원을 지정해 수업 생성 시 즉시 확정 | 현장 즉시 배정 대응 |
| 횟수 차감 시점 | PT/골프 레슨은 예약 승인이나 일정 확정 시가 아니라 수업 완료(completed) 시 차감 | 회원/강사 잔여 횟수 기준 일치 |

### 3.3 수업 서명 정책

#### 일반 PT/GX — 강사 서명만으로 수업 완료

소액(1회 3~5만원) 수업은 분쟁 빈도가 낮으므로 강사 서명만으로 완료 처리한다.

#### 골프 레슨 — 강사 + 회원 쌍방서명 필수

| 항목 | 내용 |
|------|------|
| **왜 골프만 쌍방서명인가?** | 1회 레슨 단가가 10~20만원으로 PT(3~5만원) 대비 3~5배 높다. 금액이 클수록 "수업을 안 했는데 수업료가 빠졌다"는 민원이 발생한다. |
| **실제 사례** | 골프 레슨 후 회원이 "그 날 수업을 안 받았다"고 주장 → 강사는 증빙 없음 → 수업료 환불 분쟁 |
| **해결 방식** | 양측 모두 서명해야 수업 완료 → 서명 이미지 + 타임스탬프 + 디바이스 정보 저장 → 법적 증거력 확보 |
| **서명 방식** | 대면 서명 (회원이 옆에 있을 때) 또는 원격 서명 요청 (회원이 먼저 퇴장한 경우) |
| **서명 대기 시간** | 24시간 (원격 서명 시 회원이 당일 못 할 수 있음) |
| **미서명 리마인드** | 12시간 후 재발송 (깜빡한 경우 리마인드) |
| **서명 만료** | 24시간 초과 시 서명 만료 → 강사에게 알림 → 재요청 필요 |

### 3.4 노쇼 정책

| 규칙 | 내용 | 이유 |
|------|------|------|
| 노쇼 1회 | 경고 알림 | 첫 번째는 실수일 수 있으므로 경고에 그침 |
| 노쇼 3회 누적 | 예약 제한 (센터 설정에 따라 1주/2주) | 반복 노쇼로 다른 회원의 예약 기회 침해 방지 |
| 센터별 설정 | 노쇼 페널티 기준을 센터가 직접 설정 가능 | 센터마다 엄격도가 다름 (프리미엄 센터는 더 엄격) |
| 노쇼 면제 | 강사가 수동으로 면제 처리 가능 | 비상 상황 (사고, 질병 등) 대응 |

### 3.5 이용권 정책

| 유형 | 동작 | 비고 |
|------|------|------|
| **기간제** | 시작일~종료일 내 무제한 이용 | 헬스 자유이용권 등 |
| **횟수제** | 잔여 횟수 차감 방식, 0회 되면 만료 | PT, 골프 레슨 등, 차감 시점은 수업 완료(completed) |
| **홀딩** | 기간제만 가능, 최소 7일~최대 30일 (센터 설정) | 여행, 부상 등 일시 정지 |
| **만료 알림** | D-30, D-14, D-7, D-3, D-1 자동 푸시 | 재등록 유도, FC에게도 동시 알림 |

### 3.6 결제 정책

| 항목 | 내용 | 이유 |
|------|------|------|
| 결제 수단 | 토스페이먼츠 (카드/계좌이체/간편결제) | 국내 결제 시장 점유율 + 연동 편의성 |
| 이용권 자동 생성 | 결제 완료 후 이용권 자동 생성 | 결제 후 즉시 이용 가능해야 함 |
| 환불 | 앱에서 환불 신청 → CRM에서 센터장/매니저가 처리 | 무분별한 환불 방지 (관리자 승인 필수) |
| 쿠폰 | 결제 시 1장만 적용 가능 | 중복 할인 방지 |

### 3.7 상담 정책 (FC)

| 항목 | 내용 | 이유 |
|------|------|------|
| 상담 유형 | 상담 / OT / 체험 / 재등록상담 | 유형별 통계 분석, 전환율 추적 |
| 상담 결과 | 등록 / 미등록 / 보류 (상태가 "완료"일 때만 결과 입력) | 진행 중인 상담에 결과를 잘못 기록하는 것 방지 |
| 후속 조치 필수 | 미등록/보류 시 후속 조치 필드 입력 필수 | 리드 관리 누락 방지 ("언제 다시 연락할지" 기록) |
| 상담 수정 기한 | 등록 후 7일 이내만 수정 가능 | 데이터 무결성 (오래된 상담 기록 임의 변경 방지) |

### 3.8 데이터 접근 정책

| 역할 | 접근 범위 | 권한 |
|------|----------|------|
| **회원** | 본인 데이터만 | 읽기 + 본인 정보 수정 |
| **트레이너/골프강사** | 담당 회원 데이터만 (본인이 배정된 수업의 회원) | 읽기 + 수업/체성분 기록 |
| **FC** | 담당 회원 데이터만 (담당자로 지정된 회원) | 읽기 + 상담 기록 |
| **스태프** | 전체 회원 조회 가능하되 읽기전용 | 읽기만 (수정/삭제 불가) |

> **구현 방식**: Supabase RLS(Row Level Security)로 DB 레벨에서 강제. 앱 코드에서 접근 제어를 우회할 수 없다.

## 4. 핵심 유저 저니

### 4.1 저니 1 — 신규 회원의 첫 앱 사용

```
센터 방문
  → 상담 (FC 또는 스태프)
  → 이용권 구매 (현장 결제 or 앱 결제)
  → 스태프가 CRM에 회원 등록
  → 회원에게 "앱 설치해주세요" 안내
  → 회원이 앱 설치 (App Store / Google Play)
  → 앱 연동: 이름 + 연락처 + SMS 인증
  → 비밀번호 설정
  → 홈 화면 진입
  → QR 체크인으로 첫 출석
```

> **핵심 포인트**: 회원이 스스로 가입하는 것이 아니다. CRM에 먼저 등록된 회원만 앱 연동이 가능하다. 이는 센터 소속이 확인된 회원만 앱을 사용하도록 하기 위함이다.

### 4.2 저니 2 — 회원의 하루 (PT 수업 있는 날)

```
[출근길 08:30]
  앱 열기 → 홈에서 "오늘 14:00 PT" 확인

[점심시간 12:00]
  예약 시간 변경하고 싶음
  → 예약 관리 → 14:00 수업 취소 (2시간 전이므로 취소 가능)
  → 18:00 시간대 재예약

[퇴근 후 17:50]
  헬스장 도착 → QR 체크인 (앱에서 QR 코드 표시 → 센터 기기 스캔)
  → 자동 출석 기록

[18:00~18:50]
  PT 수업 진행
  → 트레이너가 수업 완료 + 서명
  → 체성분 측정

[19:00]
  앱에서 체성분 변화 확인 (지난달 대비 체지방 -1.2%)
  → 이용권 잔여 횟수 확인 (PT 7회 남음)
```

### 4.3 저니 3 — 트레이너의 하루

```
[출근 09:00]
  앱 열기 → 오늘 수업 6개 확인 (10:00, 11:00, 14:00, 15:00, 18:00, 19:00)

[10:00 첫 수업]
  수업 시작 버튼 → PT 진행
  → 출석 체크 (참석)
  → 수업 완료 → 서명 패드에서 서명
  → 체성분 기록 (체중 72.3kg, 체지방률 18.2%)

[11:00 두번째 수업]
  회원 안 옴 → 10분 대기 → 노쇼 처리
  → 회원에게 자동 알림 발송

[14:00~19:00]
  ... 수업 반복 ...

[퇴근 20:00]
  성과 탭에서 오늘 확인:
  - 수업 완료율: 5/6 (83%)
  - 노쇼: 1건
  - 담당 회원 잔여 PT: 평균 8.2회
```

### 4.4 저니 4 — 골프 프로의 수업 완료 프로세스 (대면 서명)

```
[레슨 종료]
  앱에서 수업 완료 버튼
  → 수업 내용 확인 (날짜: 2026-04-17, 시간: 10:00~10:50, 회원: 홍길동, 내용: 드라이버 스윙 교정)

[강사 서명]
  강사 서명 패드에 서명

["대면 서명" 선택] — 회원이 옆에 있으니까
  회원에게 디바이스 전달
  → 회원이 "수업을 정상 수강했습니다" 문구 확인 후 서명

[완료]
  양측 서명 완료 → 수업 상태: completed
  → 레슨 확인서 자동 생성 (서명 이미지 + 타임스탬프 + 디바이스 정보 포함)

[월말 정산]
  레슨 확인서 목록에서 이번 달 완료된 수업 확인
  → PDF로 다운로드 → 정산 자료로 제출
```

### 4.5 저니 5 — FC의 만료 회원 재등록 흐름

```
[아침 09:00]
  FC 홈 → "만료 예정 7명" 확인

[D-7 회원 "홍길동" 탭]
  회원 상세:
  - 이용권: 3개월 헬스 자유이용, 만료 D-7
  - 출석률: 주 3회 (활성 회원)
  - 마지막 상담: 3개월 전 (신규 등록)

["재등록 상담" 등록]
  → 전화 상담 진행
  → "3개월 PT 재등록은 어떠세요?"
  → 상담 결과: 보류
  → 후속 조치: "3일 후 재상담" (필수 입력)

[3일 후 D-4]
  앱 알림: "홍길동 후속 상담 예정"
  → 재상담 진행
  → 등록 결정!
  → 상담 결과: 등록
  → 결제 안내 (앱 결제 또는 현장 결제)
```

### 4.6 저니 6 — 원격 서명 (골프, 회원이 먼저 떠난 경우)

```
[레슨 종료]
  회원이 급한 일로 먼저 퇴장

[강사 처리]
  앱에서 수업 완료 → 강사 서명
  → "원격 서명 요청" 선택

[회원 수신]
  회원 앱으로 푸시 알림:
  "[골프 레슨] 이준호 프로가 수업 완료 서명을 요청합니다"
  → 회원: 알림 탭 → 서명 화면
  → "수업을 정상 수강했습니다" 확인 → 서명 → 완료

[강사 확인]
  강사 앱에 실시간 반영: "서명 완료"

[만약 회원이 서명을 안 하면?]
  12시간 경과 → 리마인드 알림 재발송
  24시간 경과 → 서명 만료 → 강사에게 "서명 만료" 알림
  → 강사가 재요청 또는 대면 서명으로 전환 필요
```

## 8. 개발 로드맵

### Phase 1 — MVP (회원 기본)

> **비즈니스 가치**: 회원이 전화 안 하고 앱으로 예약/확인 → 프론트 업무 50% 감소

**목표**: 일반 회원이 앱으로 헬스장 핵심 서비스를 이용할 수 있는 상태

- 로그인 / 인증 (MA-001)
- 회원 홈 대시보드 (MA-100)
- QR 체크인 (MA-110)
- 출석 이력 (MA-111)
- 수업 예약 / 취소 (MA-120 ~ MA-122)
- 이용권 조회 (MA-131)
- 내 프로필 (MA-130)
- 설정 (MA-155)

### Phase 2 — 회원 확장 + 트레이너

> **비즈니스 가치**: 트레이너 현장 업무 디지털화 → 수업 기록 누락 0건, 체성분 기록 영구 보관

**목표**: 회원 기능 고도화 + 트레이너 현장 업무 지원

- 체성분 차트 (MA-132)
- 결제 이력 (MA-133)
- 온라인 상품 구매 (MA-140 ~ MA-142)
- 쿠폰함 (MA-135), 마일리지 (MA-136)
- 트레이너 홈 (MA-200)
- 수업 캘린더 (MA-210)
- 수업 시작 / 완료 / 서명 (MA-212)
- 담당 회원 관리 (MA-220 ~ MA-221)
- 체성분 / 신체정보 기록 (MA-222)
- PT 횟수 / 완강 현황 (MA-230)

### Phase 3 — 골프강사 + FC + 스태프

> **비즈니스 가치**: 골프 레슨 분쟁 제로화 (쌍방서명), FC 리드 관리 체계화 → 재등록률 향상

**목표**: 전 역할 앱 지원 완성

- 골프강사 쌍방서명 (MA-312)
- 레슨 확인서 조회 (MA-313)
- FC 상담 / 리드 관리 (MA-410 ~ MA-431)
- 스태프 출석 처리 (MA-520)
- 푸시 알림 (MA-150, MA-251, MA-451, MA-551)
- 구독 / 자동결제 (MA-154)

### Phase 4 — 고도화

> **비즈니스 가치**: 데이터 기반 운영 → KPI 자동 집계, 오프라인 환경 대응

**목표**: 운영 안정화 및 고급 기능 추가

- 운동 프로그램 배정 (MA-224)
- 운동 이력 기록 (MA-225)
- KPI / 성과 대시보드 (MA-240, MA-440)
- 오프라인 모드 (네트워크 불안정 환경 대응)
- 1:1 문의 (MA-153)
- 공지사항 (MA-152)

## 9. 성공 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 앱 연동률 | 활성 회원 80%+ | `app_linked_at` 비율 (CRM 등록 회원 중 앱 연동 완료 비율) |
| QR 체크인률 | 전체 출석의 90%+ | QR 출석 건수 / 전체 출석 건수 (수동 출석 비중 10% 이하) |
| 예약 앱 전환률 | 전화 예약 → 앱 예약 70%+ | 예약 채널 분석 (앱 예약 / 전체 예약) |
| 수업 서명률 | 100% | 서명 완료 건수 / 전체 수업 완료 건수 |
| FC 상담 기록률 | 100% | 앱 상담 기록 건수 / 실제 상담 건수 |
| 만료 회원 재등록률 | 50%+ | FC 상담 후 등록 전환 건수 / 만료 예정 회원 수 |
| 트레이너 앱 채택률 | 전체 트레이너 100% | 앱으로 수업 완료 처리하는 트레이너 비율 |
| 골프 레슨 분쟁 발생률 | 0건 | 쌍방서명 도입 후 수업료 관련 민원 건수 |

## 10. 제약사항과 가정

| 항목 | 내용 | 비고 |
|------|------|------|
| CRM 선행 필수 | CRM이 이미 운영 중인 센터가 대상 (CRM 없으면 앱 사용 불가) | 앱은 CRM의 모바일 확장 |
| 단일 센터 | 한 센터에 하나의 앱 인스턴스 (멀티 센터 전환은 Phase 4 이후) | 다중 지점 운영 센터는 지점별 별도 앱 |
| 인터넷 필수 | 인터넷 연결 필수 (오프라인 모드는 Phase 4에서 제한적 지원) | 지하 헬스장 등 Wi-Fi 환경 확인 필요 |
| 센터 경유 가입 | 회원 가입은 반드시 센터를 통해 (앱에서 자체 가입 불가) | CRM에 먼저 등록 → 앱 연동 순서 |
| 단일 역할 | 한 계정은 하나의 역할만 가짐 (멀티 역할 없음) | 트레이너이면서 회원인 경우 별도 계정 |
| 한국 시장 | 결제(토스페이먼츠), SMS 인증 등 한국 시장 전용 | 해외 확장 시 결제/인증 모듈 교체 필요 |
