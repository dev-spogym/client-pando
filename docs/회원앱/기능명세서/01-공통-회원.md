> 범위: 공통 기능 + 회원 기능

## 1. 기본 정책

### 1.1 회원가입 정책

- 회원 기본 생성은 `CRM`에서 이뤄진다.
- 앱에서는 `휴대폰 인증 + 비밀번호 설정`으로 연동을 완료한다.
- 체크리스트의 `회원가입`은 현재 기준에서 `CRM 선등록 + 모바일 앱 연동`으로 해석한다.

### 1.2 리워드 정책

- 체크리스트의 `포인트`는 `마일리지`로 통일한다.
- `포인트 사용`은 `마일리지 사용`으로 해석한다.
- `배지`는 회원앱 현재 범위에 포함한다.
- 리워드 도메인은 `쿠폰 + 마일리지 + 배지`로 정의한다.

### 1.3 결제 / 상품 정책

- PT 플랜 외에 `헬스장 이용권`, `골프장 이용권`, `골프 레슨권`, `재등록 플랜`을 같은 상품 도메인에서 다룬다.
- 회원은 `상품 스토어`와 `개인 결제` 두 경로로 결제를 시작할 수 있다.
- 결제수단은 `카드`, `계좌이체`, `네이버페이`, `카카오페이`를 제공한다.
- 실제 PG가 없어도 `결제 화면`, `영수증`, `결제 내역` 퍼블리싱은 제공한다.
- 마일리지는 결제 단계에서 차감형 리워드로 사용할 수 있다.

### 1.4 온보딩 정책

- 앱 연동 직후 또는 첫 로그인 직후 회원은 온보딩 설문을 진행할 수 있다.
- 온보딩은 `운동 목적/성향`, `통증/체형 정보`, `첫 루틴 제안` 세 단계로 구성한다.
- FMS/체형 리포트는 온보딩 정보와 측정 데이터를 함께 참고한다.

### 1.5 개인정보 / 동의 / 탈퇴 정책

- 회원은 설정에서 `약관/정책`, `동의관리`, `회원 탈퇴` 화면으로 이동할 수 있다.
- 필수 약관은 조회 전용이고, 마케팅 수신 동의만 회원이 직접 변경한다.
- 탈퇴는 앱 내 즉시 삭제가 아니라 `탈퇴 요청 접수` 흐름으로 우선 정의한다.

## 4. 주요 기능 상세

### MFN-002. 앱 가입 / 연동

| 항목 | 내용 |
|---|---|
| 목적 | CRM에 등록된 회원이 앱 계정을 연동한다 |
| 입력 | 이름(선택), 연락처, 인증번호, 비밀번호 |
| 처리 | CRM 회원 확인 → 휴대폰 인증 → 앱 비밀번호 설정 |
| 출력 | 앱 로그인 가능 상태 |
| 비고 | 공개형 자체 회원 생성이 아니라 CRM 연동 완성 단계 |

### MFN-101. 회원 홈 대시보드

| 항목 | 내용 |
|---|---|
| 목적 | 회원이 예약, 출석, 리워드, 온보딩, 공지를 빠르게 진입한다 |
| 필수 데이터 | 오늘 수업, 출석 요약, 공지 목록, 마일리지, 배지 수, 온보딩 상태 |
| 필수 액션 | PT 예약, GX 일정, Golf 예약, 리워드 센터, 알림센터, 온보딩 |
| 현재 상태 | 보강구현 완료 |

### MFN-105. 수업 예약

| 항목 | 내용 |
|---|---|
| 목적 | 회원이 수업 상세에서 예약한다 |
| 입력 | 수업 ID, 회원 ID |
| 처리 | 정원 확인 → 예약 등록 → 내 예약 상태 갱신 |
| 예외 | 정원 초과 시 대기 예약으로 유도 |

| 항목 | 내용 |

| 출력 | 최근 측정, 이력, FMS 총점, 세부 항목, 코치 메모 |
| 정책 | 온보딩 정보가 있으면 FMS 요약 정확도를 높인다 |

| 항목 | 내용 |
|---|---|

| 입력 | 회원 ID |
| 출력 | 월별 결제 목록, 결제수단, 상태, 영수증 진입 버튼 |

## 5. 현재 구현 기준 메모

- `예약/대기/후기/알림/온보딩/배지/재등록 추천`은 현재 프로젝트에서 화면과 기본 상태 흐름까지 반영했다.
- `골프 강사 예약`, `상품 스토어`, `개인 결제`, `결제하기`, `영수증`, `약관/동의/탈퇴` 화면을 퍼블리싱 기준으로 추가했다.
- 실제 PG, 실운영 예약 원장, 실운영 리워드 원장, 회원 정보 수정, 락커 만료일은 후속 연동 또는 화면 보강이 필요하다.
- 따라서 기능 상태는 `기획 포함`, `퍼블리싱 구현`, `운영 연동 후속`을 구분해서 읽어야 한다.
