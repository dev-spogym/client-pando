> 작성일: 2026-04-23
> 버전: 2.6
> 상태: 체크리스트 37개 기준 재정렬 + 골프 강사 예약/상품 결제/개인정보 관리 보강 + 회원-트레이너 예약 정책 + 회원/직원 로그인 구조 반영
> 범위: 공통 기능 + 회원 기능

## 1. 기본 정책

### 1.1 로그인 정책

- 공통 로그인 화면 `MA-001`은 `회원 로그인`과 `직원 로그인` 두 모드로 구성한다.
- 회원 로그인은 `전화번호 + 비밀번호`를 사용한다.
- 직원 로그인은 `아이디 + 비밀번호`를 사용하며, 로그인 전 `트레이너`, `FC`, `스태프` 중 직군을 먼저 선택한다.
- `golf_trainer` 계정은 로그인 화면에서는 `트레이너` 계열로 묶는다.
- 로그인 성공 후 기본 진입 경로는 `member → /`, `trainer/golf_trainer/admin → /trainer`, `fc → /fc`, `staff → /staff`다.

### 1.2 회원가입 정책

- 회원 기본 생성은 `CRM`에서 이뤄진다.
- 앱에서는 `휴대폰 인증 + 비밀번호 설정`으로 연동을 완료한다.
- 체크리스트의 `회원가입`은 현재 기준에서 `CRM 선등록 + 모바일 앱 연동`으로 해석한다.

### 1.3 리워드 정책

- 체크리스트의 `포인트`는 `마일리지`로 통일한다.
- `포인트 사용`은 `마일리지 사용`으로 해석한다.
- `배지`는 회원앱 현재 범위에 포함한다.
- 리워드 도메인은 `쿠폰 + 마일리지 + 배지`로 정의한다.

### 1.4 결제 / 상품 정책

- PT 플랜 외에 `헬스장 이용권`, `골프장 이용권`, `골프 레슨권`, `재등록 플랜`을 같은 상품 도메인에서 다룬다.
- 회원은 `상품 스토어`와 `개인 결제` 두 경로로 결제를 시작할 수 있다.
- 결제수단은 `카드`, `계좌이체`, `네이버페이`, `카카오페이`를 제공한다.
- 실제 PG가 없어도 `결제 화면`, `영수증`, `결제 내역` 퍼블리싱은 제공한다.
- 마일리지는 결제 단계에서 차감형 리워드로 사용할 수 있다.

### 1.5 온보딩 정책

- 앱 연동 직후 또는 첫 로그인 직후 회원은 온보딩 설문을 진행할 수 있다.
- 온보딩은 `운동 목적/성향`, `통증/체형 정보`, `첫 루틴 제안` 세 단계로 구성한다.
- FMS/체형 리포트는 온보딩 정보와 측정 데이터를 함께 참고한다.

### 1.6 개인정보 / 동의 / 탈퇴 정책

- 회원은 설정에서 `약관/정책`, `동의관리`, `회원 탈퇴` 화면으로 이동할 수 있다.
- 필수 약관은 조회 전용이고, 마케팅 수신 동의만 회원이 직접 변경한다.
- 탈퇴는 앱 내 즉시 삭제가 아니라 `탈퇴 요청 접수` 흐름으로 우선 정의한다.

### 1.7 수업 예약 / 승인 정책

- 회원은 공개 수업을 바로 예약할 수 있고, 트레이너의 개인 PT/골프 레슨 가능 시간을 보고 예약 요청할 수도 있다.
- 회원이 트레이너에게 직접 요청한 예약은 `트레이너 승인 대기` 상태로 생성한다.
- 트레이너가 자신의 캘린더에서 회원을 지정해 수업을 직접 생성하는 경우, 회원 추가 승인 없이 즉시 확정한다.
- 회원은 `내 예약 / 수업 이력`과 `이용권 / 잔여 회차`에서 승인 상태, PT 차감 완료 일자, 잔여 횟수, 다음 예약 예정 일자를 함께 확인할 수 있어야 한다.
- PT/골프 레슨 횟수는 예약 승인 또는 일정 확정만으로 차감되지 않고, `수업 완료(completed)` 시점에만 차감된다.
- 골프 레슨 예약은 예약 생성만 아니라 `내 골프 예약 일정` 확인까지 같은 도메인으로 본다.

## 4. 주요 기능 상세

### MFN-001. 로그인

| 항목 | 내용 |
|---|---|
| 목적 | 회원과 직원이 같은 앱에서 역할에 맞는 로그인 경로로 진입한다 |
| 입력 | 회원: 전화번호, 비밀번호 / 직원: 아이디, 비밀번호, 직원 유형(트레이너/FC/스태프) |
| 처리 | 입력값 검증 → 계정 인증 → 실제 role과 선택한 직원 유형 일치 여부 확인 → 역할별 기본 홈 이동 |
| 출력 | 회원 홈(`/`), 트레이너 홈(`/trainer`), FC 홈(`/fc`), 스태프 홈(`/staff`) |
| 비고 | `golf_trainer`는 직원 로그인 화면에서는 `트레이너` 선택으로 진입한다 |

### MFN-002. 앱 가입 / 연동

| 항목 | 내용 |
|---|---|
| 목적 | CRM에 등록된 회원이 앱 계정을 연동한다 |
| 입력 | 이름(선택), 연락처, 인증번호, 비밀번호 |
| 처리 | CRM 회원 확인 → 휴대폰 인증 → 앱 비밀번호 설정 |
| 출력 | 앱 로그인 가능 상태 |
| 비고 | 공개형 자체 회원 생성이 아니라 CRM 연동 완성 단계 |

### MFN-101. 회원 홈 대시보드

| 항목 | 내용 |
|---|---|
| 목적 | 회원이 예약, 출석, 리워드, 온보딩, 공지를 빠르게 진입한다 |
| 필수 데이터 | 오늘 수업, 출석 요약, 공지 목록, 마일리지, 배지 수, 온보딩 상태 |
| 필수 액션 | PT 예약, GX 일정, Golf 예약, 리워드 센터, 알림센터, 온보딩 |
| 현재 상태 | 보강구현 완료 |

### MFN-105. 수업 예약

| 항목 | 내용 |
|---|---|
| 목적 | 회원이 공개 수업을 예약하거나 트레이너 승인형 수업의 예약 요청을 생성한다 |
| 입력 | 수업 ID 또는 트레이너 슬롯 ID, 회원 ID, 예약 메모(선택) |
| 처리 | 예약 유형 판별 → 즉시 확정 또는 승인 대기 생성 → 내 예약 상태 갱신 |
| 예외 | 정원 초과 시 대기 예약으로 유도, 트레이너 승인 필요 수업은 확정 전까지 대기 상태 유지 |

### MFN-107. 내 예약 / 수업 이력

| 항목 | 내용 |
|---|---|
| 목적 | 회원이 예약 현황, 승인 대기 상태, 완료된 수업 이력, 골프/PT 개인 일정을 한 곳에서 확인한다 |
| 입력 | 회원 ID, 필터(전체 / 승인 대기 / 확정 / 완료 / 취소) |
| 출력 | 예약 상태, 수업 일자, 트레이너명, 수업 유형, 후기 작성 가능 여부 |
| 정책 | 예약 요청형 수업은 승인/거절 이력과 상태 변경 시간을 함께 표시한다 |
| 현재 보완 포인트 | 서비스별 이력 분리와 캘린더형 일정 표현은 후속 보강이 필요하다 |

### MFN-110. 이용권 / 잔여 회차 조회

| 항목 | 내용 |
|---|---|
| 목적 | 회원이 이용권 만료 정보뿐 아니라 PT 잔여 횟수와 최근 차감 일자, 다음 예약 예정 일자를 구분해 확인한다 |
| 입력 | 회원 ID, 이용권 ID(선택) |
| 출력 | 총 횟수, 사용 횟수, 잔여 횟수, 만료일, 최근 차감 일자, 다음 예약 예정 일자 |
| 정책 | PT / 골프 레슨처럼 횟수제 이용권은 일정 정보와 함께 보여주되, 횟수 차감은 `수업 완료` 이후 값만 반영한다 |
| 현재 보완 포인트 | 락커 만료일과 서비스별 상세 이력은 후속 보강이 필요하다 |

## 5. 현재 구현 기준 메모

- `예약/대기/후기/알림/온보딩/배지/재등록 추천`은 현재 프로젝트에서 화면과 기본 상태 흐름까지 반영했다.
- `골프 강사 예약`, `상품 스토어`, `개인 결제`, `결제하기`, `영수증`, `약관/동의/탈퇴` 화면을 퍼블리싱 기준으로 추가했다.
- `회원/직원 로그인 분기`, `트레이너 승인형 예약`, `골프 예약 일정 조회`, `PT 잔여횟수 + 차감 완료 이력`은 현재 프로젝트 기준 화면과 상태 흐름까지 반영했다.
- `서비스별 일정/이력 세분화`, 실운영 승인 이력, 실운영 캘린더 뷰 보강은 추가 후속 범위다.
- 실제 PG, 실운영 예약 원장, 실운영 리워드 원장, 회원 정보 수정, 락커 만료일은 후속 연동 또는 화면 보강이 필요하다.
- 따라서 기능 상태는 `기획 포함`, `퍼블리싱 구현`, `운영 연동 후속`을 구분해서 읽어야 한다.
