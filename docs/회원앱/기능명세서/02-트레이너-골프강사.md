## 비즈니스 맥락

### 트레이너 기능이 해결하는 문제
- **수업 캘린더**: 엑셀/종이 스케줄 → 앱 캘린더, 실시간 예약 반영, 변경 알림
- **수업 시작/완료/서명**: 수업 진행 기록을 구두 → 디지털화, 완료 증빙(서명), 출석/노쇼 자동 집계
- **체성분/신체정보 기록**: 종이/개인 노트에 흩어진 기록 → 앱에 통합, 회원에게 공유, 변화 추이 자동 차트화
- **PT 횟수/완강 현황**: "이 회원 PT 몇 회 남았지?" → 앱에서 즉시 확인, 완강 임박 시 재구매 안내 타이밍 포착
- **강사 성과 KPI**: 수동 계산 → 자동 집계 (완료율, 재구매율, 노쇼율, 만족도)

### 골프강사 쌍방서명이 필요한 이유
**문제 상황:**
- 골프 레슨은 1회 10~20만원 (PT 3~5만원의 3~5배)
- "수업을 안 했는데 수업료가 빠졌다" 민원이 실제로 발생
- 강사 혼자 서명하면 회원 입장에서 "강사가 멋대로 처리했다" 불신

**해결:**
- 강사 서명 + 회원 서명 = 양측이 "수업이 정상 진행됐다"는 합의
- 서명 이미지 + 타임스탬프 + 디바이스 정보 = 법적 증거력
- PDF 확인서 = 분쟁 시 명확한 증빙

**왜 PT는 쌍방서명이 아닌가?**
- PT 단가가 낮아 분쟁 빈도/강도가 낮음
- PT는 보통 센터 내에서 진행되어 CCTV 등 별도 증빙 가능
- 모든 수업에 쌍방서명 적용 시 업무 부담 과중 → 비용 대비 효과가 낮음

### 핵심 비즈니스 규칙 요약
| 규칙 | 내용 | 이유 |
|------|------|------|
| 수업 완료 시 서명 필수 | PT: 강사만, 골프: 양측 | 수업 진행 증빙 |
| 원격 서명 대기 24시간 | 24시간 후 만료 | 무한 대기 방지 |
| 미서명 12시간 자동 재발송 | 회원에게 알림 재발송 | 서명 누락 방지 |
| 노쇼 시 페널티 자동 부과 | 센터 설정에 따라 | 노쇼 억제 |
| 서명 거부 시 에스컬레이션 | 매니저에게 알림 | 분쟁 해결 |
| 체성분은 담당 회원만 기록 | 배정된 회원만 | 데이터 무결성 |
| 회원 요청 예약은 강사 승인 필요 | 회원이 직접 잡은 PT/골프 슬롯은 pending 상태 | 무단 예약 방지, 강사 일정 통제 |
| 강사 직접 배정 수업은 즉시 확정 | 강사가 캘린더에서 회원을 지정해 생성 | 현장 운영 즉시 반영 |
| 횟수 차감 기준 | 예약 승인/직접 배정 시가 아니라 수업 완료(completed) 시 | 회원/강사 잔여 횟수 기준 일치 |

### MFN-201. 강사 홈 대시보드

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-201 |
| 기능명 | 강사 홈 대시보드 |
| 분류 | 트레이너 |
| 대상 역할 | trainer, golf_trainer |
| 우선순위 | Critical |
| 화면 ID | MA-200 |
| CRM 연관 | 수업 관리 > 일정, 회원 관리 > 출석, 알림 센터 |

#### 기능 설명

강사가 앱 로그인 후 최초로 진입하는 메인 화면으로, 오늘의 수업 현황·출석 상태·예약 현황·미확인 알림 요약을 한눈에 확인할 수 있는 대시보드를 제공한다.

#### 상세 요구사항

- 오늘 날짜 기준 담당 수업 목록을 시간순으로 표시한다.
- 수업별 출석 현황(참석/노쇼/지각/미확인)을 상태 뱃지로 표시한다.
- 예약 대기 중인 수업 건수를 별도 카드로 노출한다.
- 미확인 알림 건수를 상단 벨 아이콘에 뱃지로 표시한다.
- 대시보드 카드는 Pull-to-Refresh로 갱신할 수 있다.
- 각 수업 카드를 탭하면 해당 수업 상세(MA-212)로 이동한다.

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 조회 날짜 | date | 자동 | 앱 기준 오늘 날짜(로컬) |
| 강사 ID | UUID | 자동 | 로그인 세션에서 추출 |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 오늘 수업 수 | integer | 전체 / 완료 / 예정 건수 |
| 출석 현황 | object | 참석 N / 노쇼 N / 지각 N / 미확인 N |
| 예약 현황 | integer | 대기 예약 건수 |
| 알림 요약 | array | 최근 5건 알림 목록 |
| 수업 목록 | array | 오늘 수업 카드 리스트 |

#### 비즈니스 규칙

- 수업 목록은 시작 시간 오름차순 정렬한다.
- 완료된 수업은 목록 하단으로 이동한다.
- 알림은 생성 시간 내림차순으로 최대 5건만 요약 표시한다.

#### 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 오늘 수업 없음 | "오늘 예정된 수업이 없습니다" Empty State 표시 |
| 네트워크 오류 | 캐시 데이터 표시 + 오프라인 배너 노출 |
| 세션 만료 | 로그인 화면으로 리디렉트 |

#### 앱 구현 사항

- Supabase Realtime 구독으로 수업 상태 변경 시 실시간 반영
- 로컬 캐시(AsyncStorage)에 마지막 성공 응답 저장, 오프라인 시 활용
- 하단 탭 바의 홈 탭에 진입 시 자동 갱신

### MFN-202. 수업 캘린더 조회

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-202 |
| 기능명 | 수업 캘린더 조회 |
| 분류 | 트레이너 |
| 대상 역할 | trainer, golf_trainer |
| 우선순위 | High |
| 화면 ID | MA-210 |
| CRM 연관 | 수업 관리 > 캘린더 |

#### 기능 설명

월·주·일 뷰 전환이 가능한 캘린더에서 담당 수업 일정을 시각적으로 확인한다. 수업 상태별 색상 코딩과 내 수업 필터를 제공하며, 회원이 요청한 예약 승인 업무도 같은 화면에서 처리한다.

#### 상세 요구사항

- 월/주/일 뷰 탭으로 전환 가능하다.
- 내 수업만 보기 토글 필터를 제공한다.
- 수업 상태별 색상 구분: scheduled(파랑), in_progress(노랑), completed(초록), cancelled(회색), no_show(빨강)
- 예약 요청 상태 구분: pending_member_request(보라), pending_trainer_confirm(주황)
- 날짜 셀 탭 시 해당 날짜의 수업 목록을 바텀 시트로 표시한다.
- 수업 블록 탭 시 수업 상세(MA-212)로 이동한다.
- 회원 요청 예약 항목에는 승인 / 거절 액션을 제공한다.
- 강사가 직접 만든 일정과 회원이 요청한 일정을 생성 주체로 구분 표시한다.

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 뷰 타입 | enum | 필수 | month / week / day |
| 기준 날짜 | date | 필수 | 기본값: 오늘 |
| 내 수업 필터 | boolean | 선택 | 기본값: true |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 수업 이벤트 목록 | array | 날짜별 수업 블록 데이터 |
| 날짜별 수업 건수 | object | 날짜 키 → 건수 값 맵 |

#### 비즈니스 규칙

- 월 뷰에서는 날짜 셀에 수업 건수 도트만 표시한다.
- 주/일 뷰에서는 시간 그리드에 수업 블록을 표시한다.
- 조회 범위는 현재 월 기준 ±3개월이다.

#### 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 해당 날짜 수업 없음 | "수업이 없습니다" 텍스트 표시 |
| 데이터 로딩 중 | 스켈레톤 UI 표시 |

#### 앱 구현 사항

- react-native-calendars 또는 커스텀 캘린더 컴포넌트 사용
- 월 변경 시 해당 월 데이터 사전 로드(prefetch)

### MFN-203. 수업 목록 조회

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-203 |
| 기능명 | 수업 목록 조회 |
| 분류 | 트레이너 |
| 대상 역할 | trainer, golf_trainer |
| 우선순위 | High |
| 화면 ID | MA-211 |
| CRM 연관 | 수업 관리 > 수업 목록 |

#### 기능 설명

담당 수업 전체를 리스트 뷰로 조회하며, 날짜 범위 및 수업 상태 필터를 적용할 수 있다.

#### 상세 요구사항

- 날짜 범위 필터(시작일~종료일) 제공한다.
- 수업 상태 필터(scheduled / in_progress / completed / cancelled) 다중 선택 가능하다.
- 수업 카드에 수업명, 시간, 참여 회원 수, 상태를 표시한다.
- 무한 스크롤(페이지당 20건)로 목록을 로드한다.
- 카드 탭 시 수업 상세(MA-212)로 이동한다.

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 시작일 | date | 선택 | 기본값: 오늘 -7일 |
| 종료일 | date | 선택 | 기본값: 오늘 +30일 |
| 상태 필터 | array | 선택 | 복수 선택 가능 |
| 페이지 번호 | integer | 자동 | 1부터 시작 |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 수업 목록 | array | 수업 카드 데이터 배열 |
| 전체 건수 | integer | 필터 조건 기준 총 수업 수 |
| 다음 페이지 여부 | boolean | 추가 로드 가능 여부 |

#### 비즈니스 규칙

- 기본 정렬: 시작 시간 내림차순(최신순)
- 취소된 수업은 기본 필터에서 제외된다.

#### 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 검색 결과 없음 | "조건에 맞는 수업이 없습니다" Empty State |
| 네트워크 오류 | 재시도 버튼 노출 |

#### 앱 구현 사항

- FlatList + onEndReached로 무한 스크롤 구현
- 필터 변경 시 목록 초기화 후 재조회

### MFN-204. 수업 시작

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-204 |
| 기능명 | 수업 시작 |
| 분류 | 트레이너 |
| 대상 역할 | trainer, golf_trainer |
| 우선순위 | Critical |
| 화면 ID | MA-212 |
| CRM 연관 | 수업 관리 > 수업 진행 |

#### 기능 설명

예약된 수업을 강사가 직접 시작 처리하여 상태를 scheduled에서 in_progress로 전환하고, 실제 시작 시간을 기록한다.

#### 상세 요구사항

- 수업 상세 화면의 "수업 시작" 버튼으로 진입한다.
- 수업 시작 가능 시간: 예약 시작 시간 기준 -15분 ~ +30분 이내
- 시작 확인 다이얼로그를 표시한 후 확정 시 상태를 변경한다.
- 시작 시각(실제)을 서버에 기록한다.
- 시작 후 출석 체크 화면(MFN-205)으로 자동 이동한다.

#### 입력 항목

### MFN-206. 수업 완료 + 서명

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-206 |
| 기능명 | 수업 완료 + 서명 |

| 우선순위 | Critical |
| 화면 ID | MA-212 |
| CRM 연관 | 수업 관리 > 수업 완료, 이용권 > 횟수 차감 |

#### 기능 설명

수업 진행을 종료하고 강사가 SignaturePad에 서명하여 수업 완료를 확정한다. 서명 이미지는 Supabase Storage에 저장된다.

#### 상세 요구사항

- "수업 완료" 버튼 탭 시 수업 내용 확인 화면을 표시한다.
- 수업 요약(날짜, 시간, 참여 회원, 출석 현황)을 확인한다.
- SignaturePad 컴포넌트로 강사 서명을 입력받는다.
- 서명 초기화(지우기) 버튼을 제공한다.
- 서명 완료 후 서명 이미지를 PNG 형식으로 Supabase Storage에 업로드한다.
- 업로드 성공 시 수업 상태를 in_progress → completed로 변경한다.

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 수업 ID | UUID | 필수 | 대상 수업 식별자 |
| 강사 서명 이미지 | binary | 필수 | PNG, 최대 2MB |
| 완료 메모 | text | 선택 | 수업 메모 |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 수업 상태 | enum | completed |
| 서명 이미지 URL | string | Supabase Storage URL |
| 완료 시각 | timestamp | 서버 기준 UTC |

#### 비즈니스 규칙

- 서명 없이 완료 불가(서명 영역 빈 경우 버튼 비활성화)
- 서명 이미지 저장 경로: lesson_signatures/{lesson_id}/trainer_{timestamp}.png
- 완료 처리 후 회원에게 수업 완료 푸시 알림 발송
- 이용권 차감은 completed 상태 전환 시점에 자동 처리
- 회원 요청 예약 승인 또는 강사 직접 일정 생성만으로는 횟수를 차감하지 않는다.

#### 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 이미지 업로드 실패 | 재시도 최대 3회, 실패 시 오류 모달 |
| 서명 영역 비어 있음 | 완료 버튼 비활성화 |
| 이미 완료된 수업 | "이미 완료된 수업입니다" 토스트 |

#### 앱 구현 사항

- react-native-signature-canvas 또는 커스텀 SVG SignaturePad 사용
- 서명 이미지 Base64 → Blob 변환 후 Supabase Storage upload
- 업로드와 상태 변경은 트랜잭션으로 처리(실패 시 롤백)

### MFN-207. 노쇼 처리

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-207 |
| 기능명 | 노쇼 처리 |

#### 기능 설명

#### 상세 요구사항

- 출석 체크에서 노쇼 선택 시 자동 연동된다.
- 노쇼 사유를 선택할 수 있다(연락 없음/사전 연락/기타).
- 페널티 부과 여부를 강사가 최종 확인한다.
- 노쇼 기록은 회원의 출석 이력에 영구 보존된다.
- CRM 관리자에게 노쇼 발생 알림을 발송한다.

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 수업 ID | UUID | 필수 | 대상 수업 |
| 회원 ID | UUID | 필수 | 노쇼 회원 |
| 노쇼 사유 | enum | 필수 | no_contact / pre_contact / other |
| 사유 메모 | text | 선택 | 기타 사유 상세 |
| 페널티 적용 여부 | boolean | 필수 | 기본값: 지점 정책 따름 |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 노쇼 기록 ID | UUID | 생성된 노쇼 기록 식별자 |
| 페널티 내용 | object | 부과된 페널티 정보 |

#### 비즈니스 규칙

- 노쇼 발생 시 해당 회원의 누적 노쇼 횟수가 증가한다.
- 누적 노쇼 3회 이상 시 CRM에 경고 플래그 자동 설정
- 페널티는 지점별 설정값(횟수 차감 또는 경고)에 따라 자동 부과

#### 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 이미 노쇼 처리된 회원 | 중복 처리 불가 안내 |
| 수업 완료 후 24시간 경과 | 노쇼 처리 불가, 관리자 문의 안내 |

#### 앱 구현 사항

- 노쇼 처리 시 CRM webhook 트리거로 관리자 알림 발송
- Supabase DB 트리거로 누적 횟수 자동 업데이트

| 항목 | 타입 | 설명 |
|---|---|---|
| 페널티 이력 목록 | array | 건별 페널티 정보 |
| 누적 페널티 수 | integer | 유형별 누적 건수 |

#### 비즈니스 규칙

- 페널티 취소는 부과 후 7일 이내만 가능하다.
- 취소 이력도 로그로 보존된다.
- 관리자 권한의 경우 7일 제한 없이 취소 가능하다.

### MFN-212. 체성분 측정 등록

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-212 |
| 기능명 | 체성분 측정 등록 |
| 분류 | 트레이너 |
| 대상 역할 | trainer, golf_trainer |
| 우선순위 | High |
| 화면 ID | MA-222 |
| CRM 연관 | 회원 관리 > 체성분 |

#### 기능 설명

회원의 체성분 측정 결과를 입력하면 BMI·BMR을 자동 계산하여 저장하는 기능이다.

#### 상세 요구사항

- 측정 항목: 체중, 골격근량, 체지방률, 내장지방 레벨
- 입력 후 BMI = 체중(kg) / 키(m)² 자동 계산
- BMR = Mifflin-St Jeor 공식 자동 계산
- 이전 측정값과의 변화량(±)을 계산하여 표시
- 측정일은 기본값 오늘, 수정 가능

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 회원 ID | UUID | 필수 | 대상 회원 |
| 측정일 | date | 필수 | 기본값: 오늘 |
| 체중 | decimal | 필수 | kg, 소수점 1자리 |
| 골격근량 | decimal | 필수 | kg, 소수점 1자리 |
| 체지방률 | decimal | 필수 | %, 소수점 1자리 |
| 내장지방 레벨 | integer | 선택 | 1~20 |
| 메모 | text | 선택 | 최대 200자 |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 기록 ID | UUID | 생성된 체성분 기록 식별자 |
| BMI | decimal | 자동 계산값 |
| BMR | decimal | 자동 계산값 (kcal/day) |
| 체중 변화량 | decimal | 이전 측정 대비 ±kg |
| 체지방률 변화량 | decimal | 이전 측정 대비 ±% |

#### 비즈니스 규칙

- BMI 계산에는 회원 프로필의 키 정보를 자동 참조한다.
- 키 정보 미등록 시 BMI 계산 불가 안내 후 신체정보 등록 유도
- 동일 날짜 중복 등록 시 덮어쓰기 여부 확인

#### 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 키 정보 없음 | BMI 필드 비활성화 + 신체정보 등록 링크 |
| 체중 범위 오류 (1~300kg 외) | 입력 필드 오류 표시 |
| 중복 날짜 | "해당 날짜에 이미 기록이 있습니다. 덮어쓰시겠습니까?" |

#### 앱 구현 사항

- 입력값 변경 시 BMI/BMR 실시간 계산 및 미리보기
- 체성분 기록 저장 후 추이 차트 데이터 자동 갱신

### MFN-223. 회원 예약 요청 승인 / 거절

| 항목 | 내용 |

| 기능명 | 회원 예약 요청 승인 / 거절 |
| 분류 | 트레이너 |
| 대상 역할 | trainer, golf_trainer |
| 우선순위 | High |
| 화면 ID | MA-210 |
| CRM 연관 | 수업 관리 > 예약 요청 |

#### 기능 설명

회원이 트레이너 또는 골프강사에게 직접 요청한 예약을 검토해 승인 또는 거절한다. 승인 시 정식 수업 일정으로 확정되고, 거절 시 회원 앱에 사유와 함께 상태가 반영된다.

#### 상세 요구사항

- 승인 대기 요청 목록을 날짜순으로 표시한다.
- 요청 카드에 회원명, 요청 시간, 서비스 유형, 메모, 잔여 횟수를 표시한다.
- `승인`, `거절`, `시간 변경 제안` 액션을 제공한다.
- 승인 시 기존 캘린더 일정으로 승격하고 회원 앱 `MA-122`에 확정 상태를 동기화한다.
- 거절 시 사유를 남기고 회원 앱에 거절 상태를 반영한다.
- 승인 또는 시간 변경 제안만으로는 이용권 횟수를 차감하지 않고, 차감은 수업 완료 시점에만 수행한다.

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 예약 요청 ID | UUID | 필수 | 승인 대상 요청 |
| 처리 결과 | enum | 필수 | approve / reject / suggest_time |
| 처리 메모 | text | 선택 | 거절 또는 시간 제안 사유 |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 처리된 예약 상태 | enum | confirmed / rejected / pending_member_response |
| 확정 수업 ID | UUID | 승인 시 생성 또는 연결된 수업 ID |

#### 비즈니스 규칙

- 회원 요청 예약만 승인/거절 대상이다.
- 강사가 직접 생성한 수업은 별도 승인 단계 없이 확정 상태여야 한다.
- 거절 또는 시간 변경 제안 처리 시 회원에게 알림을 발송한다.
- 승인 후 생성된 확정 일정은 `예정 수업`으로 관리하고, 횟수 차감은 완료 처리 후 반영한다.

#### 예외 처리

|---|---|

- 차감 횟수는 완료된 수업 기준으로만 계산하고, 예정 수업은 별도 일정 정보로 분리한다.

### MFN-301. 수업 완료 쌍방서명

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-301 |
| 기능명 | 수업 완료 쌍방서명 |
| 분류 | 골프강사 |
| 대상 역할 | golf_trainer |
| 우선순위 | Critical |
| 화면 ID | MA-312 |
| CRM 연관 | 수업 관리 > 레슨 완료, 레슨 확인서 관리 |

#### 기능 설명

골프 레슨 수업 완료 시 강사와 회원이 각각 서명하여 레슨 사실을 쌍방 확인하는 기능이다. 법적 효력을 갖는 레슨 확인서를 자동 생성하기 위해, 강사가 먼저 서명하고 회원이 대면 또는 원격 방식으로 서명을 완료한다.

---

#### 쌍방서명 전체 플로우

```
[강사 앱] 수업 완료 버튼 탭
       ↓
[1단계] 수업 내용 확인 화면 표시
  - 날짜, 시간, 회원명, 수업 내용 요약
  - "서명 진행" 버튼
       ↓
[2단계] 강사 서명 패드 표시
  - SignaturePad 입력
  - 서명 완료 버튼
       ↓
[강사 서명 저장] → 수업 상태: pending_member_sign
       ↓
[3단계] 회원 서명 방식 선택
  ┌─────────────────────────────────────┐
  │  대면 모드      │    원격 모드       │
  │  (같은 장소)   │  (회원 앱 서명)   │
  └────────┬────────┘────────┬──────────┘
           ↓                 ↓
  디바이스 회원 전달   FCM 푸시 알림 발송
  회원 서명 입력       회원 앱에서 서명
           ↓                 ↓
  [강사 앱에서 실시간 확인]
       ↓
[4단계] 양측 서명 완료
  - 수업 상태: completed
  - 레슨 확인서 자동 생성
  - 회원/강사에게 완료 알림
```

---

#### 상세 요구사항

**[1단계 수업 내용 확인]**
- 수업 날짜, 시작/종료 시각, 회원 이름, 수업 유형(레슨 분류)을 표시한다.
- 강사가 입력한 수업 메모(운동 이력, 특이사항)를 요약 표시한다.
- 잘못된 정보 발견 시 "수정" 버튼으로 수업 상세 화면으로 돌아갈 수 있다.

**[2단계 강사 서명]**
- 전체 화면 SignaturePad를 제공한다.
- 서명 필압(굵기)은 속도에 반응하여 자연스러운 필기감을 구현한다.
- "다시 서명" 버튼으로 서명 초기화 가능하다.
- 최소 서명 면적(전체 패드의 5% 이상) 미충족 시 완료 불가.
- 서명 완료 버튼 탭 시:
  - 서명 이미지를 PNG로 변환하여 Supabase Storage에 업로드
  - 업로드 성공 후 수업 상태를 `pending_member_sign`으로 변경
  - 서명 시각 타임스탬프 + 강사 디바이스 정보(OS, 앱 버전) 기록

**[3단계 회원 서명 방식 선택]**

| 구분 | 대면 모드 | 원격 모드 |
|---|---|---|
| 진입 조건 | 강사·회원이 같은 장소에 있을 때 | 강사·회원이 다른 장소에 있을 때 |
| 동작 방식 | 강사 디바이스에서 회원 서명 화면으로 전환 | 회원 앱으로 FCM 푸시 알림 발송 |
| 회원 서명 위치 | 강사의 앱 내 회원 서명 패드 | 회원 본인 앱의 서명 패드 |
| 강사 화면 | "회원 서명 중..." 대기 화면 | "회원 서명 요청 완료" 상태 표시 |
| 완료 인식 | 회원 서명 후 강사 앱 자동 전환 | Supabase Realtime 구독으로 감지 |

**대면 모드 상세:**
- 강사 앱이 회원 서명 전용 화면으로 전환된다.
- 회원 서명 완료 전까지 강사는 다른 화면으로 이동 불가(잠금 화면).
- 회원이 서명 거부 시 강사가 "서명 거부" 버튼으로 상태를 기록하고 미서명 상태로 완료할 수 있다.

**원격 모드 상세:**
- 회원 앱으로 "레슨 서명 요청" 푸시 알림 발송.
- 알림에는 수업 요약 정보(날짜, 강사명, 수업명)가 포함된다.
- 회원이 알림 탭 시 회원 앱의 서명 화면으로 딥링크 이동.
- 강사 앱은 대기 상태 표시 + 실시간 서명 완료 감지(Supabase Realtime).

**[4단계 쌍방 서명 완료]**
- 양측 서명 이미지가 모두 저장되면 수업 상태를 `completed`로 변경한다.
- 레슨 확인서(MFN-302)를 자동 생성한다.
- 강사와 회원 모두에게 완료 푸시 알림을 발송한다.
- 완료 화면에서 레슨 확인서 즉시 미리보기 또는 PDF 공유 가능.

---

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 수업 ID | UUID | 필수 | 대상 레슨 수업 식별자 |
| 강사 서명 이미지 | binary | 필수 | PNG, 최대 2MB |

---

#### 비즈니스 규칙

**서명 순서 및 상태 관리**

| 단계 | 수업 상태 | 설명 |
|---|---|---|
| 수업 진행 중 | in_progress | 수업 완료 버튼 탭 전 |
| 강사 서명 완료 | pending_member_sign | 회원 서명 대기 중 |
| 양측 서명 완료 | completed | 레슨 확인서 생성 완료 |
| 서명 거부/만료 | sign_declined / sign_expired | 예외 상태 |

- 강사 서명이 반드시 먼저 완료되어야 회원 서명 요청이 가능하다.
- 회원 서명 없이 강사 단독으로 `completed` 처리 불가.
- 수업 상태가 `in_progress`가 아닌 경우 쌍방서명 프로세스 시작 불가.

**타임아웃 및 재발송**

| 이벤트 | 시간 | 동작 |
|---|---|---|
| 회원 서명 요청 후 미서명 | 12시간 | FCM 알림 재발송 |
| 회원 서명 요청 후 미서명 | 24시간 | 서명 요청 만료(sign_expired) + 강사·CRM 알림 |
| 만료 후 처리 | - | CRM 관리자 검토 후 수동 완료 처리 가능 |

**위변조 방지**

- 서명 시점의 타임스탬프(밀리초 단위)를 서명 메타데이터에 포함한다.
- 서명 디바이스 정보(OS, 앱 버전, 기기 모델)를 기록한다.
- 서명 이미지는 SHA-256 해시값과 함께 저장한다.
- 서명 세션 ID로 강사 서명과 회원 서명을 연결한다.
- 저장 경로: `lesson_signatures/{lesson_id}/trainer_{timestamp}.png` / `lesson_signatures/{lesson_id}/member_{timestamp}.png`

**서명 이미지 저장 정책**

- Supabase Storage 버킷: `lesson_signatures` (비공개, 서명된 URL로만 접근)
- 이미지 보존 기간: 5년(법적 요건 준수)
- 이미지 포맷: PNG, 최대 2MB
- 서명 이미지 접근 권한: 해당 수업의 강사 + 회원 + 관리자만 허용

#### 예외 처리

| 상황 | 처리 방법 | 비고 |
|---|---|---|
| 강사 서명 이미지 업로드 실패 | 재시도 최대 3회 → 실패 시 오류 모달 + 재진입 안내 | 상태는 in_progress 유지 |
| 서명 이미지 크기 초과 (2MB↑) | "서명을 다시 시도해 주세요" + 자동 압축 후 재시도 | 클라이언트에서 압축 처리 |
| 최소 서명 면적 미달 | "서명이 너무 작습니다. 다시 서명해 주세요" 안내 | 완료 버튼 비활성화 |
| 회원 앱 미설치(원격 모드) | 푸시 알림 발송 실패 → SMS 대체 발송 시도 | SMS 실패 시 강사에게 알림 |
| 회원 서명 24시간 미이행 | sign_expired 상태로 변경 + 강사·관리자 알림 | CRM 수동 처리 플로우 안내 |
| 회원 서명 거부(대면 모드) | "서명 거부" 버튼 탭 → 사유 입력(선택) → sign_declined 기록 | CRM 알림 발송 |
| 원격 모드 중 네트워크 끊김 | Supabase Realtime 재연결 자동 시도(최대 5회) | 재연결 실패 시 수동 새로고침 안내 |
| 수업 상태가 in_progress가 아닌 경우 | "완료 처리할 수 없는 수업 상태입니다" + 현재 상태 안내 | 진입 차단 |
| 강사 서명 후 앱 종료/크래시 | pending_member_sign 상태 보존 → 재진입 시 3단계부터 재개 | 서버 상태 기준 복원 |
| 회원 서명 중 디바이스 교체(원격) | 서명 세션 ID로 기존 요청 연속성 보장 | 세션 만료 전까지 유효 |

#### 앱 구현 사항

**SignaturePad 구현**
- 라이브러리: react-native-signature-canvas 또는 커스텀 SVG Bezier Curve 기반
- 필압 반응: velocity 기반 선 굵기(min: 1px, max: 4px)
- 최소 서명 면적: 캔버스 전체 픽셀 중 잉크 픽셀 비율 5% 이상 검증

**서명 이미지 처리**
- 서명 완료 후 Base64 → Blob 변환 → Supabase Storage multipart upload
- SHA-256 해시는 클라이언트에서 계산 후 메타데이터로 함께 전송
- 업로드와 DB 상태 변경은 Supabase Edge Function으로 원자적 처리(트랜잭션)

**실시간 서명 감지(원격 모드)**
- 강사 앱: Supabase Realtime 채널 `lesson:{lesson_id}` 구독
- 회원 서명 완료 이벤트 수신 시 자동으로 4단계 화면으로 전환
- 채널 구독 실패 시 3초 간격 폴링으로 폴백

**대면 모드 잠금 화면**
- 강사 앱 내 회원 서명 화면: 홈 버튼/제스처 뒤로가기 차단(Android back handler)
- 회원 서명 완료 또는 거부 선택 전까지 다른 화면 이동 불가

**레슨 확인서 자동 생성**
- 쌍방 서명 완료 즉시 Supabase Edge Function 트리거로 PDF 생성
- PDF 내용: 수업 정보 + 강사 서명 이미지 + 회원 서명 이미지 + 생성 타임스탬프 + QR코드(진위 확인용)
- 생성된 PDF URL을 lesson_certificates 테이블에 기록

| CRM 연관 | 수업 관리 > 레슨 확인서 |

#### 기능 설명

쌍방 서명이 완료된 레슨 수업의 확인서 목록을 조회하고, 확인서 상세 내용을 확인하거나 PDF를 다운로드·공유할 수 있다.

#### 상세 요구사항

- 확인서 목록: 쌍방 서명 완료(completed) 수업만 표시
- 목록 카드에 수업일, 회원명, 수업명, 서명 완료 시각을 표시한다.
- 기간 필터: 이번 달 / 지난 달 / 3개월 / 직접 선택
- 확인서 상세 화면: 수업 정보 + 강사 서명 이미지 + 회원 서명 이미지 + 서명 타임스탬프
- PDF 다운로드: 디바이스 로컬 저장
- PDF 공유: 시스템 공유 시트를 통해 이메일/메신저 공유
- 서명 이미지는 Signed URL(만료 1시간)로 표시

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 강사 ID | UUID | 자동 | 로그인 세션 |
| 시작일 | date | 선택 | 기본값: 이번 달 1일 |
| 종료일 | date | 선택 | 기본값: 오늘 |
| 회원명 검색 | text | 선택 | 이름 부분 검색 |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 확인서 목록 | array | 수업별 확인서 카드 데이터 |
| 전체 건수 | integer | 필터 기준 총 확인서 수 |
| 강사 서명 이미지 URL | string | Signed URL (1시간 유효) |
| 회원 서명 이미지 URL | string | Signed URL (1시간 유효) |
| PDF URL | string | 다운로드 가능 PDF Signed URL |

#### 비즈니스 규칙

- 조회 권한: 해당 수업의 담당 강사만 조회 가능
- 서명 이미지 URL은 Signed URL(유효기간 1시간)로 제공하여 무단 접근 방지
- PDF는 요청 시점에 Supabase Edge Function에서 생성(또는 기존 캐시 반환)
- 확인서 삭제 불가(법적 보존 의무)

#### 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 확인서 없음 | "완료된 레슨 확인서가 없습니다" Empty State |
| PDF 생성 실패 | "PDF 생성 중 오류가 발생했습니다. 다시 시도해주세요" + 재시도 버튼 |
| Signed URL 만료 | 자동 재발급 (갱신 요청 후 화면 새로고침) |
| 권한 없는 확인서 접근 | 403 오류 + "접근 권한이 없습니다" |

#### 앱 구현 사항

- PDF 뷰어: react-native-pdf 또는 WebView 기반 표시
- PDF 다운로드: expo-file-system 또는 react-native-blob-util 사용
- 공유: react-native Share API로 시스템 공유 시트 호출
- 목록 정렬: 서명 완료 시각 내림차순(최신순)
- 페이징: 무한 스크롤, 페이지당 20건
