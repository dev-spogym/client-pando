## 비즈니스 맥락

### FC 기능이 해결하는 문제
- **상담 이력 관리**: 카카오톡 메모/개인 노트 → 앱에 체계적 기록, 팀 공유 가능
- **만료 예정 알림**: 수동으로 만료 회원 추적 → D-30/14/7 자동 필터, 놓치는 회원 제로
- **재등록 상담**: 만료 회원에게 즉시 전화 + 상담 결과 기록 → 재등록 전환율 추적
- **FC KPI**: 전환율, TM 완료율을 직접 계산 → 자동 집계, 목표 대비 실시간 확인
- **후속 조치 필수화**: "보류" 결과에 후속 조치 안 적으면 저장 불가 → 리드 누락 방지

### 스태프 기능이 해결하는 문제
- **모바일 회원 조회**: PC 앞에 있어야만 확인 가능 → 앱으로 어디서든 조회
- **수동 출석**: QR 인식 불가 시 빠르게 대처, 사유 기록으로 추적 가능
- **읽기전용 제한**: 스태프가 실수로 데이터 수정/삭제하는 사고 방지

### 핵심 비즈니스 규칙 요약
| 규칙 | 내용 | 이유 |
|------|------|------|
| 상담 후속 조치 필수 | 미등록/보류 시 필수 입력 | 리드 관리 누락 방지 |
| 상담 수정 7일 제한 | 등록 후 7일까지만 수정 | 데이터 무결성 |
| 스태프 쓰기 권한 없음 | 회원 정보 조회만 가능 | 데이터 오염 방지 |
| 수동 출석 사유 기록 | 선택이지만 권장 | QR 불가 사유 추적 |

### MFN-401. FC 홈 대시보드

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-401 |
| 기능명 | FC 홈 대시보드 |
| 분류 | FC |
| 대상 역할 | fc |
| 우선순위 | 높음 |
| 화면 ID | MA-400 |
| CRM 연관 | CRM 상담 관리, 회원 관리, 알림 모듈 |

#### 기능 설명

FC(Fitness Consultant)가 앱 진입 시 가장 먼저 확인하는 홈 화면. 오늘의 업무 현황을 한눈에 파악할 수 있는 KPI 요약, 당일 상담 일정, 긴급 알림을 표시한다.

#### 상세 요구사항

1. 상단 KPI 카드 영역에 4개 지표를 표시한다.
   - 오늘 상담 건수 (당일 상담 예정 + 완료 합산)
   - 담당 회원 수 (활성 상태 회원)
   - 만료 예정 수 (D-30 이내)
   - 신규 리드 수 (당일 신규 등록 리드)
2. 오늘의 상담 일정 리스트를 시간 오름차순으로 표시한다.
3. 긴급 알림 배너를 상단에 표시한다. 만료 D-7 이내 회원이 대상이다.
4. 각 KPI 카드 탭 시 해당 목록 화면으로 이동한다.
5. Pull-to-refresh로 데이터를 새로고침한다.

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 없음 (자동 조회) | — | — | 로그인 FC ID 기준 자동 집계 |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 오늘 상담 건수 | number | 당일 상담 예정 + 완료 합산 |
| 담당 회원 수 | number | 활성 상태 담당 회원 수 |
| 만료 예정 수 | number | 이용권 만료 D-30 이내 회원 수 |
| 신규 리드 수 | number | 당일 신규 등록 리드 수 |
| 상담 일정 리스트 | array | 회원명, 상담 유형, 일시, 상태 포함 |
| 긴급 알림 | array | 만료 D-7 이내 회원 목록 |

#### 비즈니스 규칙

- KPI 지표는 로그인한 FC의 담당 회원 및 상담 데이터만 집계한다.
- 긴급 알림은 만료일이 오늘 기준 D-7 이내(당일 포함)인 회원에 한해 표시한다.
- 상담 일정 리스트는 취소/노쇼 상태 건도 포함하여 표시하되 시각적으로 구분한다.

#### 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 담당 회원 없음 | "담당 회원이 없습니다" 빈 상태 UI 표시 |
| 오늘 상담 없음 | "오늘 예정된 상담이 없습니다" 안내 표시 |
| 네트워크 오류 | 캐시 데이터 표시 + 새로고침 유도 토스트 |
| 데이터 로딩 중 | Skeleton UI 표시 |

#### 앱 구현 사항

- React Native FlatList로 상담 일정 렌더링
- SWR 또는 React Query로 KPI 데이터 캐싱 (stale-while-revalidate 전략)
- 긴급 알림 배너: 조건부 렌더링, 탭 시 MFN-408(만료 예정 회원 목록)로 이동
- 앱 포그라운드 복귀 시 자동 refetch

### MFN-402. 리드/상담 예정 목록

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-402 |
| 기능명 | 리드/상담 예정 목록 |
| 분류 | FC |
| 대상 역할 | fc |
| 우선순위 | 높음 |
| 화면 ID | MA-410 |
| CRM 연관 | CRM 상담 관리 모듈 |

#### 기능 설명

FC가 담당하는 리드 및 상담 일정을 유형·상태·날짜 기준으로 필터링하여 조회한다.

#### 상세 요구사항

1. 상담 유형별 필터 탭을 제공한다: 전체 / 상담 / OT / 체험 / 재등록상담
2. 상담 상태별 필터 칩을 제공한다: 전체 / 예정 / 완료 / 취소 / 노쇼
3. 날짜 범위 필터를 제공한다 (DateRangePicker, 기본값: 오늘 기준 최근 7일).
4. 리스트 항목에 회원명, 상담 유형, 일시, 상태를 표시한다.
5. 리스트 항목 탭 시 MFN-404(상담 이력 상세)로 이동한다.
6. 우측 하단 FAB(Floating Action Button) 탭 시 MFN-403(상담 이력 등록)으로 이동한다.
7. 무한 스크롤(페이지네이션) 방식으로 로드한다.

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 상담 유형 | enum | 아니오 | 상담/OT/체험/재등록상담, 기본값: 전체 |
| 상담 상태 | enum | 아니오 | 예정/완료/취소/노쇼, 기본값: 전체 |
| 날짜 범위 | date range | 아니오 | 시작일~종료일, 기본값: 최근 7일 |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 회원명 | string | 상담 대상 회원 이름 |
| 상담 유형 | enum | 상담/OT/체험/재등록상담 |
| 상담 일시 | datetime | 예약된 상담 일시 |
| 상담 상태 | enum | 예정/완료/취소/노쇼 |
| 총 건수 | number | 필터 결과 총 건수 |

#### 비즈니스 규칙

- 로그인 FC의 담당 상담 데이터만 조회한다.
- 날짜 범위는 최대 90일로 제한한다.
- 상태 칩과 유형 탭은 동시 적용 가능하다 (AND 조건).

#### 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 검색 결과 없음 | "해당 조건의 상담 이력이 없습니다" 빈 상태 UI |
| 날짜 범위 초과 (90일+) | "최대 90일 범위까지 조회 가능합니다" 경고 토스트 |
| 페이지 로드 실패 | "데이터를 불러올 수 없습니다. 다시 시도해주세요" + 재시도 버튼 |

#### 앱 구현 사항

- 필터 상태는 Zustand 또는 로컬 상태로 관리
- 유형 탭: Segmented Control 컴포넌트 사용
- 상태 필터: 가로 스크롤 가능한 Chip 컴포넌트
- 날짜 범위: 커스텀 DateRangePicker 모달
- FlatList + onEndReached 로 무한 스크롤 구현

### MFN-403. 상담 이력 등록

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-403 |
| 기능명 | 상담 이력 등록 |
| 분류 | FC |
| 대상 역할 | fc |
| 우선순위 | 높음 |
| 화면 ID | MA-411 |
| CRM 연관 | CRM 상담 관리 모듈, 회원 관리 모듈 |

#### 기능 설명

FC가 신규 또는 기존 회원과 진행한 상담 이력을 앱에서 직접 등록한다.

#### 상세 요구사항

1. 회원 선택 시 이름/연락처 검색으로 기존 회원을 검색한다. 신규 리드는 이름 + 연락처 임시 입력 허용.
2. 상담 유형, 채널, 일시, 내용, 상태, 결과, 후속 조치를 입력한다.
3. 필수 항목 미입력 시 저장 불가 및 안내 메시지 표시.
4. 저장 성공 시 MFN-402(목록 화면)로 복귀하고 목록을 새로고침한다.
5. 저장 중 로딩 스피너를 표시하고 중복 제출을 방지한다.

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 회원 | search/text | 예 | 기존 회원 검색 또는 신규 리드 이름+연락처 |
| 상담 유형 | enum | 예 | 상담/OT/체험/재등록상담 |
| 상담 채널 | enum | 예 | 방문/전화/카카오톡/DM/SNS/기타 |
| 상담 일시 | datetime | 예 | DateTimePicker |
| 상담 내용 | textarea | 예 | 최대 2,000자 |
| 상담 상태 | enum | 예 | 예정/완료/취소/노쇼 |
| 상담 결과 | enum | 아니오 | 등록/미등록/보류 |
| 후속 조치 | textarea | 아니오 | 최대 1,000자 |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 등록 결과 | boolean | 저장 성공/실패 여부 |
| 생성된 상담 ID | string | 저장 성공 시 반환되는 상담 이력 ID |

#### 비즈니스 규칙

- 상담 일시는 현재 시각 기준 최대 30일 전까지 소급 입력 가능하다.
- 상담 상태가 "예정"인 경우 상담 결과 필드는 비활성화한다.
- 신규 리드로 등록된 상담은 CRM의 리드 목록에 자동으로 추가된다.
- 동일 회원·동일 일시의 중복 등록은 경고 후 사용자 확인 시 허용한다.

#### 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 필수 항목 미입력 | 해당 필드 하이라이트 + "필수 항목을 입력해주세요" 안내 |
| 회원 검색 결과 없음 | "검색된 회원이 없습니다. 신규 리드로 등록하시겠습니까?" 확인 다이얼로그 |
| 저장 실패 (서버 오류) | "저장에 실패했습니다. 다시 시도해주세요" 토스트, 입력값 유지 |
| 중복 상담 감지 | "동일 회원의 같은 일시 상담이 있습니다. 등록하시겠습니까?" 확인 다이얼로그 |

#### 앱 구현 사항

- 회원 검색: debounce 300ms 적용, Supabase full-text search 활용
- DateTimePicker: 플랫폼별 네이티브 피커 (iOS: DatePickerIOS, Android: DatePickerAndroid) 또는 라이브러리 통합
- 폼 상태 관리: react-hook-form + zod 스키마 유효성 검사
- 저장 버튼 이중 탭 방지: isSubmitting 상태로 disabled 처리

### MFN-404. 상담 이력 조회/수정

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-404 |
| 기능명 | 상담 이력 조회/수정 |
| 분류 | FC |
| 대상 역할 | fc |
| 우선순위 | 높음 |
| 화면 ID | MA-412 |
| CRM 연관 | CRM 상담 관리 모듈 |

#### 기능 설명

등록된 상담 이력의 상세 내용을 조회하고, 내용·결과·후속 조치를 수정하거나 삭제한다.

#### 상세 요구사항

1. 상담 이력의 모든 필드를 읽기 모드로 표시한다.
2. "수정" 버튼 탭 시 편집 모드로 전환한다.
3. 편집 가능 항목: 상담 내용, 상담 상태, 상담 결과, 후속 조치.
4. 회원, 상담 유형, 상담 채널, 상담 일시는 등록 후 수정 불가.
5. "삭제" 버튼 탭 시 확인 다이얼로그 표시 후 soft delete(deleted_at 기록) 처리.
6. 삭제된 이력은 목록에서 숨기되 CRM 어드민에서는 조회 가능.

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 상담 내용 | textarea | 예 | 최대 2,000자, 수정 가능 |
| 상담 상태 | enum | 예 | 예정/완료/취소/노쇼, 수정 가능 |
| 상담 결과 | enum | 아니오 | 등록/미등록/보류, 수정 가능 |
| 후속 조치 | textarea | 아니오 | 최대 1,000자, 수정 가능 |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 상담 이력 전체 필드 | object | 등록 시 입력된 모든 항목 |
| 등록자 | string | 상담 등록 FC 이름 |
| 등록 일시 | datetime | 이력 최초 등록 일시 |
| 최종 수정 일시 | datetime | 최근 수정 일시 |

#### 비즈니스 규칙

- 본인(로그인 FC)이 등록한 상담 이력만 수정/삭제 가능하다.
- 완료 상태의 상담은 등록 후 7일 이내에만 수정 가능하다.
- 삭제는 soft delete로 처리하며 실제 DB 레코드는 보존한다.

#### 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 타인 이력 접근 시 | "접근 권한이 없습니다" 안내 후 목록으로 복귀 |
| 수정 기한 초과 (7일+) | "수정 기한이 지났습니다. 관리자에게 문의하세요" 안내 |
| 삭제 확인 취소 | 다이얼로그 닫고 상세 화면 유지 |
| 저장 실패 | "저장에 실패했습니다. 다시 시도해주세요" 토스트 |

#### 앱 구현 사항

- 읽기/편집 모드 전환: 동일 화면 내 상태 토글
- RLS 정책으로 타인 데이터 접근 자체를 차단
- 삭제: Supabase의 deleted_at 컬럼 업데이트 (실물 데이터 유지)

### MFN-406. 회원 상세 조회

FC가 담당 회원의 상세 정보를 탭 구조로 조회한다. 기본정보·이용권·출석이력·상담이력·결제이력 5개 탭으로 구성된다.

#### 상세 요구사항

1. 탭 구성: 기본정보 / 이용권 / 출석이력 / 상담이력 / 결제이력
2. 기본정보 탭: 이름, 성별, 생년월일, 연락처, 가입일, 담당 FC, 메모 요약
3. 이용권 탭: 현재 이용권명, 시작일, 종료일, 잔여 횟수, 이용권 이력 목록

5. 상담이력 탭: 상담 이력 목록 (최신순), 탭 시 MFN-404로 이동
6. 결제이력 탭: 결제 일시, 상품명, 금액, 결제 수단 목록
7. 상단 고정 헤더에 회원명, 상태 배지, 이용권 만료 D-day 표시
8. 우측 하단 FAB로 상담 이력 등록(MFN-403) 바로 진입 가능

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 회원 ID | string | 예 | 라우트 파라미터로 전달 |

| 이용권 | 현재 이용권명, 시작일, 종료일, 잔여 횟수, 이력 |

| 결제이력 | 결제 일시, 상품명, 금액, 결제 수단 |

#### 비즈니스 규칙

- FC는 담당 회원의 모든 탭을 조회 가능하다.
- 결제이력 탭의 카드번호는 마지막 4자리만 표시한다.
- FC는 조회만 가능하며 수정 권한은 없다 (메모 제외, MFN-407 참조).

#### 예외 처리

| 상황 | 처리 방법 |

| 담당 아닌 회원 접근 | RLS로 차단, "접근 권한이 없습니다" 안내 |

- Tab Navigator (react-navigation Tab) 또는 커스텀 탭 헤더 사용
- 탭별 lazy loading: 탭 전환 시 해당 탭 데이터 최초 1회 fetch
- 결제이력 카드번호 마스킹: 서버 응답에서 마스킹된 값 반환

- 메모 유형별 배지 색상 구분 (일반: 회색, 주의: 빨간색, VIP: 금색, 기타: 파란색)
- 낙관적 업데이트(Optimistic Update)로 UX 향상

| 이용권명 | string | 만료 예정 이용권 이름 |
| 만료일 | date | 이용권 만료 날짜 |
| 잔여 D-day | number | 만료까지 남은 일수 |

- 이미 만료된 회원(D-day 0 이하)은 별도 "만료됨" 섹션으로 분리하여 하단에 표시한다.
- 이용권이 여러 개인 경우 가장 빠른 만료일 기준으로 표시한다.

#### 예외 처리

#### 기능 설명

만료 예정 또는 만료된 회원을 대상으로 재등록 상담을 등록한다. 상담 유형이 "재등록상담"으로 고정되며, 이전 이용권 정보와 재등록 제안 상품을 함께 관리한다.

#### 상세 요구사항

3. 재등록 제안 상품을 드롭다운으로 선택한다 (CRM 상품 목록 연동).
4. 할인/쿠폰 적용 메모를 텍스트로 입력한다.
5. 나머지 필드(채널, 일시, 내용, 상태, 결과, 후속 조치)는 MFN-403과 동일하게 입력.
6. 저장 성공 시 CRM 재등록 상담 목록에 자동 반영.

#### 입력 항목

| 상담 유형 | enum | 예 | 재등록상담 (고정) |
| 재등록 제안 상품 | select | 아니오 | CRM 상품 목록 드롭다운 |

| 상담 채널 | enum | 예 | 방문/전화/카카오톡/DM/SNS/기타 |
| 상담 일시 | datetime | 예 | DateTimePicker |
| 상담 내용 | textarea | 예 | 최대 2,000자 |
| 상담 상태 | enum | 예 | 예정/완료/취소/노쇼 |
| 상담 결과 | enum | 아니오 | 등록/미등록/보류 |
| 후속 조치 | textarea | 아니오 | 최대 1,000자 |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 이전 이용권명 | string | 회원의 현재/최근 이용권 이름 (읽기전용) |
| 이전 이용권 만료일 | date | 현재/최근 이용권 만료일 (읽기전용) |
| 등록 결과 | boolean | 저장 성공/실패 여부 |

#### 비즈니스 규칙

- 재등록 상담은 MFN-403의 상담 이력과 동일한 테이블에 저장하되 유형으로 구분한다.

### MFN-410. FC 성과/KPI 조회

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-410 |
| 기능명 | FC 성과/KPI 조회 |
| 분류 | FC |
| 대상 역할 | fc |
| 우선순위 | 중간 |
| 화면 ID | MA-440 |
| CRM 연관 | CRM KPI 관리 모듈 |

#### 기능 설명

FC 본인의 영업 성과 KPI를 조회한다. 상담 전환율, TM 완료율, 재등록 전환율, 리드 누락률을 목표 대비로 시각화한다.

#### 상세 요구사항

1. 기간 필터를 제공한다: 이번 달 / 지난 달 / 최근 3개월 / 직접 선택
2. KPI 카드 4종을 표시한다.
   - 신규 상담 전환율: (등록 결과 = "등록" 건수 / 전체 상담 건수) × 100, 목표 35%+
   - TM 완료율: (완료 상태 상담 건수 / 예정 상담 건수) × 100, 목표 90%+
   - 재등록 전환율: (재등록상담 중 등록 결과 건수 / 전체 재등록상담 건수) × 100, 목표 50%+
   - 리드 누락률: (노쇼 건수 / 전체 리드 건수) × 100, 목표 3% 이하
3. 각 KPI 카드에 목표 달성 여부를 색상으로 구분한다 (달성: 초록, 미달: 빨간).
4. 월별 추이 차트(꺾은선 그래프)를 하단에 표시한다.

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 기간 | enum/date range | 아니오 | 이번 달/지난 달/최근 3개월/직접 선택, 기본값: 이번 달 |

#### 출력 항목

| KPI 항목 | 산출식 | 목표 |
|---|---|---|
| 신규 상담 전환율 | 등록 건수 / 전체 상담 건수 × 100 | 35%+ |
| TM 완료율 | 완료 상담 / 예정 상담 × 100 | 90%+ |
| 재등록 전환율 | 재등록 등록 건수 / 재등록상담 × 100 | 50%+ |
| 리드 누락률 | 노쇼 건수 / 전체 리드 건수 × 100 | 3% 이하 |
| 월별 추이 | 월별 각 KPI 수치 | — |

#### 비즈니스 규칙

- 본인 KPI만 조회 가능하다 (타 FC 비교 불가).
- KPI 데이터는 일 1회 서버에서 집계하여 캐싱한다 (실시간 아님).
- 데이터 없는 기간은 0으로 표시한다.

#### 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 해당 기간 데이터 없음 | "해당 기간의 데이터가 없습니다" 안내 + 0% 표시 |
| 차트 렌더링 오류 | 차트 영역에 "차트를 표시할 수 없습니다" 메시지 |

#### 앱 구현 사항

- 차트 라이브러리: `react-native-chart-kit` 또는 `victory-native`
- KPI 카드: 목표 대비 달성률 프로그레스 바 포함
- 캐시 만료: 24시간 단위, 앱 포그라운드 복귀 시 만료 여부 확인

### MFN-411. 알림 조회

| 항목 | 내용 |
|---|---|
| 기능 ID | MFN-411 |
| 기능명 | 알림 조회 |
| 분류 | FC |
| 대상 역할 | fc |
| 우선순위 | 중간 |
| 화면 ID | MA-451 |
| CRM 연관 | CRM 알림 모듈 |

#### 기능 설명

FC에게 발송된 앱 푸시 알림 이력을 조회한다. 상담 예정 알림, 이용권 만료 알림, 시스템 알림을 통합하여 표시한다.

#### 상세 요구사항

1. 알림 유형별 필터 탭을 제공한다: 전체 / 상담 예정 / 만료 알림 / 시스템
2. 알림 목록을 최신순으로 표시한다.
3. 읽지 않은 알림은 배지 또는 점으로 구분 표시한다.
4. 알림 항목 탭 시 해당 기능 화면으로 이동한다.
   - 상담 예정 알림 → MFN-402(상담 목록)
   - 만료 알림 → MFN-408(만료 예정 목록)
   - 시스템 알림 → 상세 내용 모달
5. "모두 읽음 처리" 버튼 제공.
6. 알림은 최대 90일치를 보관한다.

#### 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 알림 유형 | enum | 아니오 | 상담 예정/만료 알림/시스템, 기본값: 전체 |

#### 출력 항목

| 항목 | 타입 | 설명 |
|---|---|---|
| 알림 제목 | string | 알림 요약 제목 |
| 알림 내용 | string | 알림 상세 내용 |
| 알림 유형 | enum | 상담 예정/만료 알림/시스템 |
| 발송 일시 | datetime | 알림 발송 시각 |
| 읽음 여부 | boolean | 읽음/읽지 않음 |

#### 비즈니스 규칙

- 90일 이상 경과한 알림은 자동 삭제한다.
- 앱 아이콘 배지 수는 읽지 않은 알림 수로 표시한다.
- 알림 탭 진입 시 화면 내 모든 알림을 자동으로 "읽음" 처리하지 않고 항목별 탭 시 읽음 처리한다.

#### 예외 처리

| 상황 | 처리 방법 |
|---|---|
| 알림 없음 | "알림이 없습니다" 빈 상태 UI |
| 90일 이상 이력 조회 시도 | 자동으로 90일 이내만 표시 |

#### 앱 구현 사항

- FCM(Firebase Cloud Messaging) 기반 푸시 알림 수신
- 알림 수신 시 로컬 DB(SQLite 또는 AsyncStorage)에 저장
- 배지 수: `@react-native-community/push-notification-ios` (iOS) / 네이티브 모듈 (Android)

- 로그아웃: Supabase signOut() 호출 후 AsyncStorage 전체 삭제 및 루트 화면으로 리셋

|---|---|---|

- 쓰기 UI (FAB, 수정 버튼 등) 역할 기반 조건부 렌더링으로 숨김

### MFN-503. 회원 상세 조회 (읽기전용)

#### 상세 요구사항

1. 탭 구성: 기본정보 / 이용권 / 출석이력 / 결제이력

4. 출석이력 탭: 날짜별 출석 기록, 월별 요약 (읽기전용)
5. 결제이력 탭: 결제 일시, 상품명, 금액, 결제 수단 (읽기전용)

7. 모든 탭에서 수정/삭제 버튼을 표시하지 않는다.

| 항목 | 타입 | 필수 | 설명 |

| 결제이력 | 결제 일시, 상품명, 금액, 결제 수단 |

#### 비즈니스 규칙

- 스태프는 소속 지점 모든 회원의 상세를 조회할 수 있다.

- 결제이력 카드번호는 마지막 4자리만 표시한다.
- 수정 API 호출은 RLS에서 완전 차단한다.

#### 예외 처리

| 상황 | 처리 방법 |

| 타 지점 회원 접근 | RLS 차단, "접근 권한이 없습니다" 안내 |
| 탭 데이터 로드 실패 | 해당 탭 오류 메시지 + 재시도 버튼 |

- 수정 버튼: 역할 기반 조건부 렌더링으로 숨김

### 인증/보안

- **인증 방식**: Supabase Auth 기반 JWT 토큰 인증

- **역할 관리**: 사용자 프로필 테이블의 role 컬럼으로 fc / staff 구분
- **RLS (Row Level Security)**: Supabase RLS 정책으로 역할별 데이터 접근 제어. 앱 레벨 권한 제어와 DB 레벨 제어를 이중으로 적용
- **민감 데이터 암호화**: 결제 정보(카드번호 등)는 서버에서 암호화 저장, 응답 시 마스킹 처리
- **세션 만료**: 비활동 30일 후 자동 로그아웃
- **SSL/TLS**: 모든 API 통신은 HTTPS 필수

---

### 오프라인 처리

- **로컬 캐싱**: 기본 데이터는 AsyncStorage 또는 MMKV로 로컬 캐싱
- **캐시 유효 기간**: 회원 목록 30분, KPI 데이터 24시간, 알림 7일
- **오프라인 안내**: 네트워크 미연결 시 "네트워크 연결이 필요합니다. 캐시 데이터를 표시합니다" 배너 표시
- **오프라인 제한 기능**: 출석 처리, 상담 등록, 메모 등록은 오프라인 시 비활성화
- **서명 데이터 동기화**: 서명 데이터는 오프라인 시 로컬 저장 → 온라인 복귀 시 자동 동기화 (현재 미구현, 추후 적용 예정)
- **연결 복귀 감지**: NetInfo 라이브러리로 연결 상태 모니터링, 복귀 시 자동 refetch

### 데이터 격리

각 역할은 접근 가능한 데이터 범위가 엄격하게 제한된다.

| 역할 | 데이터 접근 범위 |
|---|---|
| member | 본인 데이터만 조회 (이용권, 출석, 결제, 수업 예약) |
| trainer | 담당 회원 + 본인 수업 데이터 |
| golf_trainer | 담당 회원 + 본인 수업 데이터 |
| fc | 담당 회원 + 본인 상담 데이터 |
| staff | 소속 지점 전체 회원 조회 (읽기전용) |
