# FitGenie 회원앱 화면설계서 — 공통 · 회원

> 작성일: 2026-04-23
> 버전: 2.6
> 문서 성격: 기획 기준 + 현재 구현 상태 동시 관리
> 기준: `회원앱리스트_2025_1228_요약.xlsx`, `APP-IA.md`, 현재 프로젝트 `src/*`

---

## 1. 문서 사용 원칙

- 이 문서는 `화면 기획 기준 문서`다.
- 각 화면에는 현재 프로젝트 기준 상태를 함께 적는다.
- `구현`, `보강구현`, `부분구현`, `후속` 네 가지 상태를 사용한다.
- 체크리스트 항목이 빠지지 않도록 `화면 ID ↔ 체크리스트` 연결을 명시한다.
- 실운영 연동이 없어도 회원 사용 흐름을 설명할 수 있는 퍼블리싱 화면은 현재 범위에 포함한다.

|---|---|---|---|---|---|
| MA-001 | 로그인 | 회원 로그인 / 직원 로그인 분기 | 회원 전화번호 로그인, 직원 아이디 로그인, 직원 유형 선택(트레이너/FC/스태프) | 구현 | `/login`, `src/screens/Login.tsx` |
| MA-002 | 앱 가입 / 연동 | CRM 회원의 앱 연동 완료 | 이름/전화 확인, 휴대폰 인증, 비밀번호 설정 | 보강구현 | `/register`, `src/screens/Register.tsx` |

### 공통 정책 메모

- 회원 기본 생성은 CRM에서 이뤄진다.
- 앱에서는 `휴대폰 인증 기반 연동`으로 가입을 완료한다.
- 공개형 즉시 회원 생성은 현재 기본 시나리오가 아니다.
- 로그인은 `회원 로그인`과 `직원 로그인`으로 나뉜다.
- 직원 로그인은 `트레이너`, `FC`, `스태프` 직군 선택 후 아이디/비밀번호로 인증한다.
- `golf_trainer` 계정은 로그인 화면에서는 `트레이너` 계열로 묶어 진입한다.
- 회원은 트레이너의 가능 시간을 보고 PT/골프 레슨을 요청할 수 있다.
- 회원이 직접 만든 트레이너 대상 예약은 `트레이너 승인 대기`를 기본 상태로 한다.
- 트레이너가 자신의 스케줄에서 회원을 직접 지정해 일정을 만들면 즉시 확정으로 처리한다.

## 4. 핵심 화면 설계 요약

### 4.0 MA-001 로그인

**목적**

- 회원과 직원이 동일 앱 안에서 본인 역할에 맞는 로그인 경로로 진입한다.

**필수 요소**

- 상단 탭: 회원 로그인 / 직원 로그인
- 회원 입력: 전화번호, 비밀번호
- 직원 입력: 직원 유형(트레이너/FC/스태프), 아이디, 비밀번호
- 로그인 성공 후 역할별 기본 홈 이동
- 회원 전용 `앱 연동하기` 링크

**현재 구현 메모**

- 로그인 화면은 `회원 로그인 / 직원 로그인` 구조로 구현됐다.
- 직원 로그인 안에서 `트레이너`, `FC`, `스태프`를 선택할 수 있다.
- `golf_trainer` 계정은 로그인 화면에서는 `트레이너` 계열로 묶어 진입한다.
- 선택한 직원 유형과 실제 계정 role이 다르면 로그인되지 않는다.

### 4.1 MA-100 홈

**목적**

- 회원이 가장 자주 쓰는 `예약`, `출석`, `리워드`, `온보딩`, `공지`를 한 화면에서 본다.

**필수 요소**

- PT 예약 바로가기
- GX 스케줄 바로가기
- Golf 예약 바로가기
- 마일리지/배지 요약 카드
- 공지/이벤트 배너
- 오늘 수업 요약
- 이번 달 출석 요약
- 온보딩 또는 추천 루틴 카드

**현재 구현 메모**

- 위 필수 요소를 홈에 모두 배치했다.
- `오늘 수업`은 현재 지점 수업 기반이라 운영 원장 연동은 추가 보완이 필요하다.

### 4.2 MA-121 수업 상세 / 예약

**목적**

- 예약 여부, 정원 현황, 강사 진입, 대기 등록 진입을 한 곳에서 처리한다.

**필수 요소**

- 수업명, 시간, 장소, 강사
- 정원 / 잔여석
- 예약하기 / 예약취소
- 트레이너 승인 필요 시 예약 요청 상태
- 대기열 등록
- 강사 상세 보기

**현재 구현 메모**

- 예약하기/예약취소/대기열 등록 버튼을 현재 프로젝트에 반영했다.
- 대기열은 로컬 상태 기반으로 관리한다.
- 트레이너 승인형 예약 상태와 즉시 예약형 수업 상태를 구분해 표시한다.

### 4.2A MA-122 내 예약 / 수업 이력

**목적**

- 회원이 내 수업 일정, 골프 예약 일정, 승인 대기 상태, 완료 이력을 한 화면에서 확인한다.

**필수 요소**

- 서비스 필터: 전체 / PT / GX / Golf
- 상태 필터: 승인 대기 / 확정 / 완료 / 취소
- 리스트 또는 캘린더 전환
- 트레이너명 / 수업일 / 시간 / 상태
- 후기 작성 진입

**현재 구현 메모**

- 예약 현황과 이력 조회 흐름은 현재 문서 범위에 포함한다.
- 승인 또는 예약 확정만으로는 횟수가 차감되지 않고, 완료 수업만 차감 이력에 반영한다.
- 서비스별 일정 분리, 승인 대기 전용 필터, 달력형 표현은 후속 보강이 필요하다.

### 4.3 MA-123 Golf 예약 / 골프 강사 예약 / 타석 선택

**목적**

- 골프 회원이 `강사 레슨 예약`과 `타석 예약`을 하나의 화면 안에서 처리한다.

**필수 요소**

- 골프 강사 예약 탭
- 강사명 / 레슨명 / 시간대 / 타석 / 가격
- 내 골프 레슨 예약 목록 / 일정
- 골프 타석 예약 탭
- 타석 상태 표시, 만석 시 대기 등록
- 레슨권 / 이용권 구매 CTA

**현재 구현 메모**

- 골프 강사 예약과 타석 예약을 한 화면에 병합했다.
- 실제 스케줄 원장 대신 퍼블리싱 기준 슬롯과 예약 목록을 제공한다.
- 회원 본인의 골프 예약 리스트와 예약 흐름은 현재 화면에 포함하며, 달력형 확장 뷰는 후속 보강 범위다.

### 4.4 MA-124 대기 예약 관리

**목적**

- 회원이 대기 순번과 자동 확정 상태를 직접 확인하고 취소한다.

**필수 요소**

- 수업명, 일시, 강사
- 현재 순번
- 자동 확정 여부
- 대기 취소

### 4.5 MA-125 강사 상세

**목적**

- 예약 전 강사의 전문분야, 후기 요약, 가능 수업을 확인한다.

**필수 요소**

- 강사명 / 경력
- 전문분야 태그
- 최근 후기 요약
- 예약 가능한 시간대 요약
- 예약 요청 CTA

### 4.6 MA-126 수업 후기 / 만족도

**목적**

- 완료 수업에 대해 별점, 태그, 한줄 후기, 추천 의향을 받는다.

**필수 요소**

- 별점 1~5
- 빠른 태그
- 한줄 후기
- NPS 점수

### 4.7 MA-127 ~ MA-128 온보딩

**목적**

- 체크리스트의 빠진 항목이었던 `운동 목적/성향`, `통증/체형 정보`, `첫 루틴 제안`을 연결한다.

**필수 요소**

- 운동 목적 복수 선택
- 운동 성향 / 강도 선택
- 통증 부위 선택
- 체형/집중 부위 선택
- 선호 요일/운동 시간
- 요약 및 첫 루틴 제안

### 4.8 MA-132 체성분 / FMS

**목적**

- 인바디와 FMS/체형 정보를 한 도메인 안에서 본다.

**필수 요소**

- 최근 체성분 요약
- 측정 이력
- FMS 총점
- 세부 항목 점수
- 코치 메모

### 4.8A MA-131 이용권 / 잔여 회차

**목적**

- 회원이 이용권 만료일뿐 아니라 PT 잔여 횟수와 차감 이력, 다음 예약 예정 일자를 구분해 확인한다.

**필수 요소**

- 이용권명 / 만료일
- 총 횟수 / 사용 횟수 / 잔여 횟수
- 최근 차감 일자
- 다음 예약 예정 일자(참고 정보)
- 차감 기준 안내: 수업 완료 후 횟수 차감
- 예약 화면 이동 CTA

**현재 구현 메모**

- 회원권/잔여 회차/만료일은 현재 구조에 포함한다.
- PT 차감 이력과 다음 예약 예정 일자를 수업 완료 기준으로 구분해 표시한다.

### 4.9 MA-133 ~ MA-142 구매 / 결제

**목적**

- PT 플랜뿐 아니라 헬스장 이용권, 골프장 이용권, 골프 레슨권, 개인 결제를 하나의 구매 흐름으로 연결한다.

**필수 요소**

- 결제 내역 목록
- 영수증 진입
- 상품 스토어 카테고리
- 상품 상세 가격/혜택/구성
- 결제수단 선택
- 마일리지 사용
- 개인 결제 직접 입력
- 영수증 상세

**현재 구현 메모**

- `/shop → /shop/:productId → /checkout/:productId → /payments/:paymentId` 흐름을 추가했다.
- `/payment/personal`에서 수동 결제 항목을 생성해 같은 결제 화면으로 연결한다.
- 결제는 퍼블리싱 기준 모의 흐름이며 실제 PG 연동은 후속이다.

### 4.10 MA-136 ~ MA-137 리워드 센터

**목적**

- 체크리스트의 `포인트/배지`를 `마일리지/배지` 구조로 정리한다.

**필수 요소**

- 쿠폰 탭
- 마일리지 탭
- 배지 탭
- 마일리지 정책
- 배지 획득 조건

### 4.11 MA-138 재등록 추천 플랜

**목적**

- 회원이 만료 전후에 재등록 후보 상품 3종을 비교한다.

**필수 요소**

- 추천 플랜 3종
- 각 플랜 가격/혜택
- 결제 CTA
- 결제/영수증 진입 연계

### 4.12 MA-150 알림센터

**목적**

- 예약, 이용권, 리워드, 공지 알림을 한곳에 모으고 액션 화면으로 이동시킨다.

**필수 요소**

- 전체/미읽음 탭
- 알림 카테고리
- 액션 버튼
- 대상 화면으로 이동

### 4.13 MA-155 ~ MA-158 설정 / 개인정보

**목적**

- 알림 설정과 함께 개인정보 관련 화면을 하나의 허브로 묶는다.

**필수 요소**

- 예약/이용권/결제/공지/마케팅 알림 ON/OFF
- 약관 / 정책 진입
- 동의관리 진입
- 개인 결제 / 상품 스토어 진입
- 앱 버전
- 회원 탈퇴
- 로그아웃

**현재 구현 메모**

- 설정 안에서 약관/정책, 동의관리, 개인 결제, 상품 스토어, 탈퇴 요청으로 바로 진입하도록 정리했다.
- 탈퇴는 즉시 삭제가 아니라 요청 접수형 퍼블리싱으로 정의했다.

## 6. 현재 구현 기준 참고 메모

- 현재 프로젝트는 `Supabase 조회 + 로컬 상태` 조합으로 일부 기능을 먼저 흡수했다.
- 대기 예약, 알림센터, 후기, 배지, 온보딩, 재등록 추천, 골프 강사 예약, 상품 결제, 약관/동의/탈퇴는 이번 보강 구현 범위다.
- 회원/직원 로그인 분기, 골프 예약 일정 조회, 트레이너 승인형 예약, PT 잔여횟수/차감 완료 이력 표시는 현재 프로젝트 기준 화면과 상태 흐름까지 반영했다.
- 실제 PG 결제, 운영 수준 골프/수업 예약 원장, 회원 정보 수정, 락커 만료일, 서비스별 이력 세분화는 후속 보강이 필요하다.
