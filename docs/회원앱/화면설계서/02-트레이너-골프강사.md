# 스포짐 회원앱 화면설계서 — 트레이너 · 골프강사

> 작성일: 2026-04-23
> 버전: 2.0
> 기준 코드: `app/(mobile)/trainer/*`, `src/screens/trainer/*`, `src/components/TrainerTabBar.tsx`, `src/components/TrainerRoute.tsx`, `src/stores/authStore.ts`
> 문서 목적: 트레이너/골프강사 영역의 현재 모바일 웹앱 화면 구조와 로그인 진입 기준 정리
> 역할 체계: `trainer`, `golf_trainer`, `admin`

## 1. 문서 기준

- 트레이너와 골프강사는 공통 로그인 화면 `MA-001`의 `직원 로그인 > 트레이너` 경로로 진입한다.
- 실제 계정 role이 `trainer`, `golf_trainer`, `admin` 중 하나일 때 `/trainer` 계열 라우트를 허용한다.
- 트레이너 계열 사용자가 회원 전용 라우트에 접근하면 역할별 기본 홈(`/trainer`)으로 되돌린다.
- 골프강사는 트레이너 화면을 공통으로 사용하며, 쌍방서명/확인서 화면을 추가로 사용한다.

### 1.1 트레이너 탭 구조

| 탭 | 경로 | 비고 |
|---|---|---|
| 홈 | `/trainer` | 강사 홈 대시보드 |
| 회원 | `/trainer/members` | 담당 회원/회원 목록 |
| 일정 | `/trainer/schedule` | 승인 요청, 직접 배정, 일정 생성 |
| 피드백 | `/trainer/feedback` | 수업별 피드백 입력 |
| MY | `/profile` | 트레이너 계열 사용자만 노출 |

### 1.2 공통 동작 규칙

- `TrainerRoute`는 `trainer`, `golf_trainer`, `admin`만 허용한다.
- `Profile`의 트레이너 계열 MY 메뉴에서 수업 목록, 노쇼/페널티, KPI, 확인서, 알림, 설정 화면으로 이동한다.
- `/trainer/members/:id`, `/trainer/classes/:id`, `/trainer/certificates/:id` 등 상세 경로에서는 하단 탭을 숨긴다.

## 3. 핵심 화면 설계 요약

### SCR-MA-200 트레이너 홈

- 목적: 오늘 수업, 방문 현황, 빠른 메뉴를 첫 화면에서 확인한다.
- 핵심 요소:
  - 강사명/역할 헤더
  - `오늘 수업`, `오늘 방문` 요약 카드
  - 다음 수업 카드
  - 회원관리 / 일정관리 / 운동피드백 / 공지조회 퀵메뉴
- 구현 메모:
  - `오늘 방문`은 담당 회원 한정이 아니라 지점 전체 출석 기준이다.
  - 직원 로그인으로 들어온 트레이너/골프강사/관리자는 같은 홈 구조를 사용한다.

### SCR-MA-210 일정 관리

- 목적: 주간 일정 조회, 회원 요청 승인/거절, 회원 직접 지정 생성, 수업 완료 흐름을 처리한다.
- 핵심 요소:
  - 주간 이동 헤더와 요일 선택
  - 선택 날짜 수업 리스트
  - 승인 대기 요청 리스트
  - 회원 지정 수업 추가 바텀시트
  - 수업 완료 후 횟수 차감 정책 안내
- 구현 메모:
  - 회원이 만든 요청은 `승인 대기` 상태로 관리한다.
  - 트레이너가 직접 회원을 지정해 생성한 일정은 즉시 확정한다.
  - PT/골프 레슨 횟수 차감은 승인 시점이 아니라 `수업 완료` 시점에 반영한다.

### SCR-MA-211 / MA-212 수업 목록 / 상세

- 목적: 강사가 오늘 이후 수업을 카드형 리스트로 보고, 상세 화면에서 완료·노쇼·쌍방서명 흐름으로 진입한다.
- 핵심 요소:
  - 수업 유형, 시간, 상태 배지
  - 회원 정보, 진행 상태, 템플릿/페널티/확인서 진입
  - 골프 수업일 경우 `쌍방서명 진행` CTA
- 구현 메모:
  - 일반 PT/GX는 완료 처리 중심
  - 골프 수업은 쌍방서명 화면으로 분기한다

### SCR-MA-213 노쇼 / 페널티

- 목적: 노쇼 처리 결과와 페널티 상태를 관리한다.
- 핵심 요소:
  - 노쇼 건수 요약
  - 회원별 페널티 카드
  - 면제/관리 액션
- 구현 메모:
  - 현재 모바일 웹앱에서는 mock/local 상태를 기준으로 화면을 제공한다.
  - 실운영 규칙은 `기능명세서/02-트레이너-골프강사.md`의 노쇼 정책과 연결한다.

### SCR-MA-220 / MA-221 회원 관리 / 상세

- 목적: 트레이너가 회원 검색, 수업 이력, 체성분, 평가, 메모, 잔여 횟수를 확인한다.
- 핵심 요소:
  - 목록: 검색, 필터, 회원 카드
  - 상세: 운동기록 / 체성분 / 평가 / 메모 + 예약/차감 이력
- 구현 메모:
  - 회원 상세에는 `수업 완료 기준 차감` 정책이 반영된 잔여 횟수와 차감 이력을 노출한다.
  - 승인 이력 전용 필터, 기간 필터 세분화는 추가 후속 범위다.

### SCR-MA-240 / MA-251 / MA-252 KPI / 알림 / 설정

- 목적: 강사 운영 지표, 역할 알림, 개인 설정을 모바일에서 확인한다.
- 핵심 요소:
  - KPI 카드: 완료율, 노쇼, 활성 회원 수
  - 알림: 예약 요청, 서명, 시스템 공지
  - 설정: 푸시, 예약 알림, 시스템 알림
- 구현 메모:
  - 설정은 현재 트레이너 전용 토글 중심으로 구성한다.
  - 로그아웃은 트레이너 MY(`/profile`)에서 수행한다.

### SCR-MA-312 / MA-313 골프강사 확장

- 목적: 골프강사의 고가 레슨 증빙 흐름을 모바일에서 완결한다.
- 핵심 요소:
  - 강사 서명 + 회원 서명
  - 확인서 목록과 확인서 상세
  - 증빙 문구와 타임라인
- 구현 메모:
  - `golf_trainer`는 로그인 구조상 별도 탭을 두지 않고 `트레이너` 계열로 진입한다.
  - 라우트는 `/trainer/*`를 공통 사용하되, 골프 수업 타입에서 쌍방서명/확인서 메뉴가 활성화된다.
