# 스포짐 회원앱 화면설계서 — FC · 스태프 (구현 상태)

> 작성일: 2026-04-22
> 기준 코드: `src/App.tsx`, `src/stores/authStore.ts`
> 문서 목적: 현재 저장소 기준 FC/스태프 구현 여부 명시
> 상태: 미구현

---

## 1. 결론

- 현재 프로젝트에는 FC 화면과 스태프 화면이 구현되어 있지 않다.
- 기존 기획 문서에 존재하던 FC/스태프 화면설계서는 현재 코드와 불일치하여, 본 문서는 구현 상태 문서로 대체한다.

## 2. 근거

### 2.1 역할 지원 범위

- `src/stores/authStore.ts`의 실제 역할 분기는 `member`, `trainer`, `admin`뿐이다.
- `fc`, `staff`, `golf_trainer` 전용 인증 분기나 프로필 타입이 없다.

### 2.2 라우트 지원 범위

- `src/App.tsx`에는 `/fc`, `/staff` 계열 라우트가 없다.
- `TrainerRoute`, `PrivateRoute` 외에 FC/스태프 전용 라우트 래퍼가 없다.

### 2.3 페이지 컴포넌트 범위

- `src/pages` 하위에 FC 화면 디렉터리나 스태프 화면 디렉터리가 없다.
- 현재 구현된 역할별 화면은 회원 화면과 `src/pages/trainer/*` 트레이너 화면뿐이다.

---

## 3. 미구현 화면 범위

| 역할 | 기획 화면 ID 범위 | 현재 상태 |
|---|---|---|
| FC | MA-400 ~ MA-452 | 미구현 |
| 스태프 | MA-500 ~ MA-552 | 미구현 |

### FC 미구현 항목

- FC 홈
- 리드/상담 예정 목록
- 상담 이력 등록/조회
- 담당 회원 목록/상세
- 만료 예정 회원 관리
- FC 성과/KPI
- FC 전용 알림/설정

### 스태프 미구현 항목

- 스태프 홈
- 회원 조회/상세
- 수동 출석 처리
- 수업 일정 조회
- 스태프 전용 알림/설정

## 4. 향후 구현 시 필요한 최소 조건

- `authStore`에 `fc`, `staff` 역할 분기 추가
- `App.tsx`에 역할별 라우트와 기본 진입 경로 추가
- 하단 탭 구조 정의
- FC/스태프 전용 페이지 컴포넌트 추가
- 데이터 접근 범위와 Supabase RLS 정책 정리
