> 문서 목적: FC/스태프 영역의 현재 모바일 웹앱 화면 구조와 로그인 진입 기준 정리

## 1. 문서 기준

- FC와 스태프는 공통 로그인 화면 `MA-001`의 `직원 로그인`에서 각각 `FC`, `스태프` 직군을 선택해 진입한다.
- 실제 계정 role이 선택한 직원 유형과 같아야 각 전용 라우트(`/fc`, `/staff`)를 허용한다.
- 직원 라우트는 실제 로그인 계정과 `preview=1&role=...` 퍼블리싱 모드 둘 다 허용한다.
- FC와 스태프는 하단 탭 구조가 서로 다르며, 상세 화면에서는 일부 탭을 숨긴다.

### 1.1 FC 탭 구조

| 탭 | 경로 | 비고 |
|---|---|---|
| 홈 | `/fc` | KPI 요약, 상담 일정 |
| 상담 | `/fc/leads` | 상담 목록 / 등록 / 상세 |
| 회원 | `/fc/members` | 담당 회원 목록 / 상세 / 메모 |
| KPI | `/fc/kpi` | 성과 대시보드 |
| 설정 | `/fc/settings` | 푸시 설정 + 로그아웃 |

### 1.2 스태프 탭 구조

| 탭 | 경로 | 비고 |
|---|---|---|
| 홈 | `/staff` | 출석/시설 요약 |
| 회원 | `/staff/members` | 읽기 전용 회원 조회 |
| 출석 | `/staff/attendance/manual` | 수동 출석 처리 |
| 일정 | `/staff/schedule` | 읽기 전용 일정 조회 |
| 설정 | `/staff/settings` | 알림 설정 + 로그아웃 |

## 3. FC 핵심 화면 설계 요약

### SCR-MA-400 FC 홈

- 목적: 오늘 상담, 담당 회원, 만료 예정, 긴급 알림을 한 화면에서 본다.
- 핵심 요소:
  - FC 이름 헤더
  - `오늘 상담`, `담당 회원`, `만료 예정`, `신규 리드` 요약 카드
  - 오늘의 상담 일정
  - 긴급 알림 카드
- 구현 메모:
  - 실제 로그인한 FC 이름을 우선 표시한다.
  - KPI 값과 목록 데이터는 현재 mock/local 상태를 기준으로 제공한다.

### SCR-MA-410 ~ MA-412 상담 영역

- 목적: 상담 목록 조회, 신규 상담 등록, 상세 확인/수정을 한 흐름으로 연결한다.
- 핵심 요소:
  - 목록: 상태/유형별 상담 리스트
  - 등록: 회원 선택, 상담 유형, 상태, 메모
  - 상세: 상담 결과와 후속 조치 확인
- 구현 메모:
  - 현재 모바일 웹앱은 mock 데이터 기반이지만 실제 경로와 화면 전환 구조를 모두 제공한다.

### SCR-MA-420 ~ MA-422 회원 영역

- 목적: FC가 담당 회원을 보고 상세 및 메모를 관리한다.
- 핵심 요소:
  - 담당 회원 리스트
  - 회원 상세
  - 회원 상세 내부 메모 시트
- 구현 메모:
  - 메모는 현재 mock/local 상태에 저장한다.
  - 회원 상세 구조는 공통 member detail 패턴을 재사용한다.

### SCR-MA-430 / MA-431 / MA-440 / MA-451 / MA-452

- 목적: 만료 예정 회원 추적, 재등록 상담, KPI, 알림, 설정을 모바일 업무 흐름으로 묶는다.
- 핵심 요소:
  - 만료 예정 회원 목록
  - 재등록 상담 입력
  - KPI 카드
  - 역할 알림
  - 설정 토글 + 로그아웃
- 구현 메모:
  - FC 설정 화면에서 로그아웃을 직접 수행할 수 있다.

## 5. 스태프 핵심 화면 설계 요약

### SCR-MA-500 스태프 홈

- 목적: 출석/락커 등 프론트 운영 지표와 빠른 실행 버튼을 제공한다.
- 핵심 요소:
  - 스태프 이름 헤더
  - 오늘 출석, 현재 이용중, 락커 사용, 지점 요약 카드
  - 회원 조회 / 수동 출석 / 일정 조회 퀵메뉴
- 구현 메모:
  - 실제 로그인한 스태프 이름을 우선 표시한다.
  - 요약 수치는 현재 mock/local 상태를 기준으로 제공한다.

### SCR-MA-510 / MA-511 회원 조회 / 상세

- 목적: 프론트 데스크에서 회원 상태를 읽기 전용으로 빠르게 확인한다.
- 핵심 요소:
  - 검색 가능한 회원 목록
  - 읽기 전용 상세
  - 결제/상담 수정 버튼 미노출
- 구현 메모:
  - 스태프 화면은 쓰기 권한 없이 조회 중심으로 설계한다.

### SCR-MA-520 수동 출석

- 목적: QR 오류 등 예외 상황에서 스태프가 직접 출석을 처리한다.
- 핵심 요소:
  - 회원 검색
  - 입장 / 퇴장 선택
  - 수동 출석 사유 입력
- 구현 메모:
  - 현재 모바일 웹앱에서는 mock/local 상태로 처리한다.

### SCR-MA-530 / MA-551 / MA-552

- 목적: 읽기 전용 일정 조회, 역할 알림, 스태프 설정을 제공한다.
- 핵심 요소:
  - 일정 카드 / 상태 표시
  - 역할 알림 리스트
  - 설정 토글 + 로그아웃
- 구현 메모:
  - 스태프 설정 화면에서 로그아웃을 직접 수행할 수 있다.

## 6. 공통 구현 메모

- FC/스태프 라우트는 `PreviewRoleRoute`를 통해 preview 모드와 실제 로그인 모드를 함께 허용한다.
- FC/스태프 상세 화면 경로에서는 하단 탭을 숨겨 집중도 높은 상세 뷰를 제공한다.
- 현재 FC/스태프 업무 데이터는 mock/local 상태를 우선 사용하지만, 라우트 구조와 탭 흐름은 실운영 기획 기준과 동일하게 유지한다.
